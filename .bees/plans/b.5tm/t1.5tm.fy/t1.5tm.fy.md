## Spec divergence note required (added during /bees-breakdown-epic of Epic B, 2026-05-03)

When updating `docs/sdd.md` `### Feature: Ephemeral-Agent Orchestration` `**Cold-start hybrid (warm vs fresh Agents)**` paragraph, this Epic must add a short paragraph documenting an **intentional divergence from the original SDD's warm-Agent intent**:

- The original SDD prescribed warm Engineer/Test Writer Agents via `Agent(name=<task-id>)` + subsequent `SendMessage` across Subtasks within a Task.
- During Epic B's planning, the official Claude Code sub-agents docs were re-read (after Epic A's probe results landed) and confirmed that **`SendMessage` is only available when `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`** (sub-agents docs, "Resume subagents" section). Removing Agent Teams (the whole point of this Bee) therefore makes the warm-Agent pattern unavailable.
- The shipped implementation uses **per-Subtask cold dispatch** for all three implementer roles (Engineer, Test Writer, Doc Writer): each Subtask spawns a fresh Agent. Coherence across dependent Subtasks is preserved through the diff (each later Engineer reads the prior Engineer's commit), matching the hub-and-spoke "diff is the handoff" pattern already documented in the SDD. Concurrent specialist work is preserved: Engineer-Subtask-N+1 runs in parallel with Test-Writer-Subtask-N once the orchestrator's reconciliation loop ticks past Engineer-Subtask-N's completion.
- Token cost is bounded by Claude Code's prompt cache (5-min TTL) — subsequent Subtasks within a Task that re-read the same files mostly hit cache.
- This divergence is tracked by a follow-up Issue (filed separately during the Epic B breakdown's "Offer Next Steps") that re-probes whether `SendMessage` becomes available without `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` in a future Claude Code version. The Issue is intended to stay open under "periodic re-test" semantics — closed only when (a) probe passes and a follow-up Bee implements warm-Agent dispatch as a token-cost optimization, or (b) we explicitly decide to drop the optimization permanently.

The added paragraph in the SDD must reference the follow-up Issue ID so a future maintainer reading the SDD finds the action path. The paragraph should be placed within the existing `**Cold-start hybrid (warm vs fresh Agents)**` block (currently lines ~188-194 of the post-b.51d SDD) — replacing the warm-Engineer/Test-Writer prose with the cold-per-Subtask reality, keeping the Doc Writer / Reviewer / PM cold-fresh prose unchanged.
