## What this Epic delivers

Rewrite `skills/bees-execute/SKILL.md` to use the reconciliation-loop pattern with background `Agent` invocations against the subagent types defined in Epic A. Drop all Agent Teams primitives, the message-flow choreography, the `blocked_on` signal, the graduated-escalation ladder, and the helper-script references. Probe recursive delegation early; pick flat or nested orchestration based on the result.

## Acceptance criteria

1. `bees-execute/SKILL.md` no longer contains any of: `TeamCreate`, `TeamDelete`, `Agent Teams`, `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS`, `force_clean_team`, `check_agent_teams`, `teammateMode`, `metadata.blocked_on`, "graduated escalation", "Team-lead message-flow choreography".
2. Skill prose references the new subagent types by name (`subagent_type: "engineer"`, etc.) rather than inlining role instructions.
3. The reconciliation-loop tick is documented: read state (bees ticket query + TaskList) → reconcile (dispatch ready, persist completed, advance) → yield. Explicit guidance: no `/loop`, no `ScheduleWakeup`, no `CronCreate`, no polling.
4. Warm/cold cold-start guidance documented:
   - Engineer + Test Writer use named Agents + SendMessage across Subtasks within a Task
   - PM, Code Reviewer, Doc Reviewer, Test Reviewer always fresh
   - Doc Writer fresh by default with opt-in warm for multi-doc-Subtask Tasks
   - Task scope end: orchestrator sends "complete" SendMessage to each warm Agent so it returns and exits
5. TaskList-as-progress-UI documented: orchestrator creates one TaskList task per concurrent Agent invocation; status transitions (`pending → in_progress → completed`) and `metadata.activity` are updated as Agents start/finish.
6. **Recursive-delegation probe completed and recorded.** Skill prose includes either:
   - (a) An Epic-level sub-orchestrator pattern that fires `Agent(subagent_type=epic-orchestrator)` for each Epic, or
   - (b) Flat orchestration with documented Epic-boundary context-clear discipline (the existing pattern, kept).
   Probe result is recorded in this Epic's done-status notes.
7. **Demo:** `/bees-execute` runs end-to-end on a small real or stand-in Bee (1-2 Epics, 2-3 Tasks, 4-6 Subtasks total) and completes without manual prods. Concurrent specialist work is observable in the TaskList UI (e.g., Engineer Subtask 1 + Test Writer Subtask 0 running in parallel).

## Implementation notes

- **Probe shape.** The recursive-delegation probe is a real `Agent` invocation that internally calls `Agent` again — minimal test, measure success or specific failure mode. If subagents cannot spawn further subagents, fall back to flat. PRD/SDD already acknowledge both outcomes.
- **Hub-and-spoke preserved.** Workers do not communicate with each other. Engineer's commits become Test Writer's input via git diff; Test Writer's output becomes PM's input via diff + ticket state. No SendMessage between workers.
- **Cross-Task / cross-Epic interaction checks.** PM responsibility (per current SKILL.md lines ~377-382). Move into the PM subagent's body in Epic A; invoke from this skill's orchestrator at the appropriate tick.
- **Time-bounded review iteration** (~10 work items / ~5 turns short-circuit). Preserve in the PM subagent body.
- **Worktree isolation prompt** (current lines ~110-122) preserved as orchestrator-direct AskUserQuestion — no Agent involved.
- **One commit per Task** (current Section 4.1) preserved — orchestrator-direct, no Agent.
- **Inter-Epic interaction checkpoint** (current Section 4.2 ~lines 442-454) preserved — orchestrator-direct, no Agent.
