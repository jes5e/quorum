## What Needs to Change

This is a research/observation Subtask. No source files are modified.

- Identify or author a small Bee suitable for the demo: 1-2 Epics, 2-3 Tasks, 4-6 Subtasks total, with Bee body and ticket bodies populated to the standard `/bees-breakdown-epic` outputs.
- Run `/bees-execute` against the demo Bee in an isolated context (worktree if available; otherwise a feature branch — the worktree-vs-branch isolation prompt is preserved by Task 1).
- Capture the following observations into a notes artifact (a local notes file, or directly into the Subtask's status updates and final report — the executing Engineer's choice):
  - **Start-to-finish completion.** The skill ran from invocation to Final Output without manual prods. If any prod was required, record what and why.
  - **Concurrent specialist work in TaskList.** At least one observation where two or more implementer Agents (Engineer / Test Writer / Doc Writer in any combination) had overlapping `in_progress` status in the TaskList UI. Capture a description or screenshot reference.
  - **Per-Task commits.** Each Task produced exactly one git commit per the Section 4.1 commit procedure. Record the commit count and verify it matches the Task count.
  - **PM Scoped-marker dispatch.** During per-Task PM review, the orchestrator passed the helper path placeholder to the PM Agent per Subtask 2.2's wiring. If the demo Bee has no Scoped-marker on the Bee body, record "n/a — demo Bee has no Scoped-marker" instead. Either is a valid finding for the artifact.
  - **Post-completion fresh-eyes sweep.** Section 6's `Agent(subagent_type=general-purpose, run_in_background=true)` returned a finding (or "no issues found"). Record the result.
  - **Cross-OS coverage.** The host platform on which the demo ran (POSIX or Windows). If only one platform was tested, that gap is recorded honestly in Subtask 7.2.
- Do NOT modify Epic 8s's body in this Subtask — that artifact is written by Subtask 7.2.
