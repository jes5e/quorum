## Context

Section 5 of `skills/bees-execute/SKILL.md` ("Final Bee-level Code, Doc and Eng reviews") forms a new review Team and runs the iteration loop for review feedback. With substrate switch, this becomes three concurrent ephemeral `Agent(subagent_type=..., run_in_background=true)` invocations against `code-reviewer`, `test-reviewer`, `doc-reviewer` subagent types Epic A landed.

## What Needs to Change

- Remove the opening `TeamDelete` call on the last Epic's team and its `force_clean_team.py` fallback.
- Replace "Form a new review Team... Use bee-scoped agent names..." with: dispatch three concurrent ephemeral Agents using subagent types `code-reviewer`, `test-reviewer`, `doc-reviewer`. Use TaskList tasks `code-reviewer-<bee-id>`, `test-reviewer-<bee-id>`, `doc-reviewer-<bee-id>` matching Task 2's naming convention but bee-scoped.
- Reframe conditional spawn logic: "If you invoked the Engineer in the first team, invoke the Code Reviewer" → "If Engineer Agents were dispatched during the Epic loop, dispatch the code-reviewer Agent now." Same for Test Writer/test-reviewer and Doc Writer/doc-reviewer.
- Replace inline Instructions blocks for Code Reviewer / Test Reviewer / Doc Reviewer with one-line role summaries and pointers to `agents/<role>.md`.
- Update iteration loop: "Reform or re-use the first team to do the work" → "If feedback requires action, dispatch fresh ephemeral implementer Agents per Section 3's dispatch shape (Engineer/Test Writer/Doc Writer/PM as needed). Stay in delegate mode."
- Preserve the "may choose to **NOT** spawn the Product Manager" optional-PM-on-iteration logic.
- Preserve "Note: This could create an infinite loop so you may ignore feedback so long as you present it in Final Review." UX contract.
- Preserve "**IMPORTANT** Stay in delegate mode and do not do the work yourself." UX contract.
- Remove all references to: `TeamCreate`, `TeamDelete`, `force_clean_team.py`, "team-scoped agent names" (in the Agent-naming sense), "Reform the implementation team... using the just-disbanded pattern".

## Why

Final Bee review is one of the cross-cutting work-to-preserve items. The three reviewer roles + iteration loop semantics are load-bearing. Only the Team-formation ceremony is obsolete.

## Cross-Task Contract

Final Section-rewrite Task. Builds on Task 2's dispatch shape. After Task 5, Section 5 has no Agent Teams residue.
