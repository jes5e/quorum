## Context

The current `skills/bees-execute/SKILL.md` precondition section hard-fails if `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` is not set, and references the bundled helper `check_agent_teams.py` for verification. With the substrate switch to ephemeral background `Agent` invocations against the seven custom subagent types Epic A landed under `agents/`, this precondition is obsolete and must be replaced with a check that the seven required subagent types are registered in the running Claude Code session.

Per Epic A's `## Probe results — verified update` finding: custom subagents are loaded by Claude Code at session start only — mid-session installs require restart or `/agents` to hot-reload. So a missing subagent_type registration at skill invocation almost always indicates a missing post-install restart or a missing install altogether. The hard-fail must guide the user to (a) verify the install per `README.md` `## Install`, and (b) restart Claude Code or use the `/agents` command to hot-reload.

## What Needs to Change

- Remove the `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` precondition bullet and the rationale prose ("The skill spawns a team unconditionally...").
- Remove the "Verifying the Agent Teams precondition" subsection and both OS-paired `check_agent_teams.py` snippets.
- Add a new precondition bullet listing the seven subagent types required: `engineer`, `test-writer`, `doc-writer`, `pm`, `code-reviewer`, `test-reviewer`, `doc-reviewer`.
- Add a new "Verifying the subagents precondition" subsection. Hard-fail message must direct the user to verify the install per `README.md` `## Install` AND restart Claude Code or use `/agents`. Do NOT silently fall back to `general-purpose` or skip the dispatch.
- If any `teammateMode` or 'Display backend' references exist in Section 1 (Preconditions), remove them. (Likely no such occurrences today, but belt-and-braces against Agent-Teams residue leaking in via this rewrite.)
- Preserve all other preconditions: Plans hive colonized; CLAUDE.md `## Documentation Locations` section; CLAUDE.md `## Build Commands` section with all five required keys.
- Do NOT delete `check_agent_teams.py` from disk in this Task — that's Epic E's scope. Just remove the SKILL.md reference.

## Why

The substrate switch from Agent Teams to ephemeral background Agents makes the Agent Teams precondition obsolete. The new precondition surfaces missing-restart symptoms at run-time, before any dispatch attempts.

## Cross-Task Contract

This Task establishes that ALL subsequent dispatches in Tasks 2-7 of this Epic assume the seven subagent types are registered. Tasks 2-7 must NOT add fallback logic for missing subagent types — the precondition gate is the single point of enforcement.

The precondition is a **procedural gate**: the hard-fail message directs the user to restart Claude Code or run `/agents`, but the skill itself does not programmatically introspect the subagent registry (other than the optional belt-and-braces `claude agents` shell-out described in Subtask 1.1 and gated on Engineer research). Downstream Tasks 2-7 may treat "precondition passed" as "dispatch is the right thing to attempt" and MUST NOT add per-dispatch fallback logic for missing `subagent_type`. If a late dispatch failure does occur, the orchestrator surfaces it to the user with the same restart guidance — it does not silently fall back to `general-purpose` or skip the dispatch.

The remaining `force_clean_team.py` references in Sections 3, 4.2, and 5 are intentionally **out of Task 1's scope**; they are removed by Tasks 2 (Section 3), 4 (Section 4.2), and 5 (Section 5) downstream in this Epic. Task 1 covers only the Preconditions section's `check_agent_teams.py` references.
