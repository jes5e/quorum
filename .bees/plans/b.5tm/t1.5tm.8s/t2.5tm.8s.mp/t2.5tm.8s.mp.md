## Context

Section 3 of `skills/bees-execute/SKILL.md` is the orchestration heart — currently 250+ lines of Agent Teams team-formation, message-flow choreography, blocked_on signal handling, graduated-escalation ladder, pre-dispatch state checks, and inline role-Instructions blocks for Engineer / Test Writer / Doc Writer / PM. Epic 8s replaces all of that with a reconciliation-loop pattern over ephemeral background `Agent` invocations against the seven custom subagent types Epic A landed in `agents/`.

The reconciliation-loop tick is event-driven — wake on Agent completion notifications + user input. NO `/loop`, NO `ScheduleWakeup`, NO `CronCreate`, NO polling. Each tick: read state (bees ticket query + TaskList) → reconcile (dispatch ready Subtasks via fresh per-Subtask `Agent(subagent_type=<role>, run_in_background=true)`; persist completed; advance to PM at Task-done; advance to inter-Epic checkpoint at Epic-done) → yield.

Per-Subtask cold dispatch for Engineer/Test Writer/Doc Writer. NO `Agent(name=...)` warming, NO `SendMessage` between Subtasks (gated on Agent Teams per the official Claude Code sub-agents docs). Reviewers and PM always fresh per invocation. Hub-and-spoke preserved via substrate.

Recursive delegation: NOT supported per official sub-agents docs ("Subagents cannot spawn other subagents"). Skill ships flat orchestration with documented Epic-boundary context-clear discipline; cite docs in rewritten prose so future maintainers don't re-litigate.

TaskList is the progress UI: orchestrator creates one TaskList task per concurrent Agent invocation; status transitions update as Agents start/finish; `metadata.activity` carries finer progress.
