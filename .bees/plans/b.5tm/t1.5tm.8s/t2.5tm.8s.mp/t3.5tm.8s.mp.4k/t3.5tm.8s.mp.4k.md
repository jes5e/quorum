## Context

This Subtask is the first half of Task 2 in Epic t1.5tm.8s. Section 3 of `skills/quo-execute/SKILL.md` currently runs ~250 lines of Agent Teams team-formation, message-flow choreography, blocked_on signal handling, graduated-escalation ladder, pre-dispatch state checks, and inline role-Instructions blocks for Engineer / Test Writer / Doc Writer / PM. Epic 8s replaces all of that with a reconciliation-loop pattern over ephemeral background `Agent` invocations against the seven custom subagent types Epic A landed in `agents/`.

Carrying-forward architectural decisions:

- **Per-Subtask cold dispatch** for Engineer / Test Writer / Doc Writer. Each Subtask spawns a fresh `Agent(subagent_type=<role>, run_in_background=true)`. NO `Agent(name=...)` warming, NO SendMessage between Subtasks. PM and reviewers always fresh.
- **Reconciliation-loop driven by Agent completion notifications.** Explicit callout: NO `/loop`, NO `ScheduleWakeup`, NO `CronCreate`, NO polling.
- **Flat orchestration.** No recursive delegation per official sub-agents docs ("Subagents cannot spawn other subagents").
- **Hub-and-spoke preserved.** Workers do not communicate with each other; the diff is the handoff. No peer-to-peer messaging, no `metadata.blocked_on`, no graduated-escalation ladder.
- **Subagent types referenced by name.** No inline role-prose blocks lifted into SKILL.md — Epic A already moved those into `agents/<role>.md`.

This Subtask establishes the Section 3 skeleton; Subtask 2.2 layers the TaskList progress-UI prose with naming convention and the Scoped-marker PM dispatch wiring on top.

The "Testing discipline — avoid redundant full-workspace runs" and "Running long commands" subsections are preserved verbatim — they survive the substrate change unchanged.
