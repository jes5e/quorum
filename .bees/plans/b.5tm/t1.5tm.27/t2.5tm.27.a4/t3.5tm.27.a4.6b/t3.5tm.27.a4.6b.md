## Context

This subtask depends on T3.S1 (which established `subagents/pm.md` with the placeholder for marker prose). The PM subagent is invoked from two execution skills with **different marker-handling semantics**, both of which must coexist in a single subagent body:

1. **bees-execute path (Grandparent-Bee marker check):** the Grandparent Bee is canonical and known to the orchestrator. The PM checks the Bee body for a marker. On helper exit 2 (malformed marker, missing doc, missing heading), the PM **hard-fails the spec review** — never silent-fallback to the full doc.
2. **bees-fix-issue path (`up_dependencies`-based opportunistic discovery):** Issues live in the `issues` hive and have no canonical parent-Plan-Bee field; the PM iterates the Issue's `up_dependencies` (a deliberate dual-use of that field as both blocker validation and optional scope-context discovery) to find a Plan Bee in the `plans` hive that may carry a marker. Discovery is best-effort: missing markers, malformed markers, non-Plans-hive entries, or helper hard-fails fall back to full-doc spec content (with the helper's stderr surfaced to the user on hard-fail).

The error-handling asymmetry between the two paths (hard-fail vs. best-effort fallback) is intentional and load-bearing — it MUST be preserved verbatim.

The pm.md body REFERENCES `docs/doc-writing-guide.md` `## The Scoped-marker contract` as the source of truth for marker grammar rather than re-deriving the parsing rules.

**Helper-resolution-path strategy (locked inter-Epic contract):** Inside a subagent body, "this skill's base directory" is not the right primitive — subagents are invoked by `subagent_type` and do not have a sibling-skill resolution model the way an installed skill does. The strategy is:

- **Primary**: orchestrator passes the resolved absolute path in the Agent invocation prompt as a named placeholder. pm.md uses the placeholder verbatim. Sibling Epics B and C MUST implement this.
- **Fallback** (defensive, for isolated testing): pm.md self-resolves at canonical install location — `~/.claude/skills/bees-breakdown-epic/scripts/scoped_marker_resolver.py` (global), then `<repo>/.claude/skills/bees-breakdown-epic/scripts/scoped_marker_resolver.py` (per-project). Body emits a one-line warning when the fallback is used.
