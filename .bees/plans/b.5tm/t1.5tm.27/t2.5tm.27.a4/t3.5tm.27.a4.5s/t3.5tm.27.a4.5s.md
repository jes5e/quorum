## Context

The PM subagent is the most prose-heavy of the seven role files because it owns spec traceability, cross-Task / cross-Epic interaction checks, the time-budget short-circuit on review-skill loops, and orchestrates in-flight `/bees-code-review` and `/bees-doc-review` invocations during per-Task review. Per CLAUDE.md `## Model assignment in execution skills`, the PM's runtime model is user-chosen (Opus or Sonnet) at the start of a `/bees-execute` or `/bees-fix-issue` run, so the file ships with `model: opus` as the shipped default and downstream Epics B/C handle the runtime override mechanism — same pattern as T1.S3 (`doc-writer.md`) and T2.S3 (`doc-reviewer.md`). The Skill tool is in the allowlist so the body can dispatch `/bees-code-review` and `/bees-doc-review` during the per-Task review pass. The Write tool is in the allowlist so T3.S2 can use the temp-file pattern the Scoped-marker helper consumes.

This subtask establishes the file with frontmatter and lifts the general PM Instructions; T3.S2 layers in the Scoped-marker integration prose afterward.
