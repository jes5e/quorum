## Context

The README's `## Install` section currently documents two install patterns — Option A (global, recommended for single-user machines) and Option B (per-project) — each with paired POSIX bash and Windows PowerShell snippets. Each pattern's snippet has one line that copies `skills/*` into `~/.claude/skills/` (Option A) or `<repo>/.claude/skills/` (Option B). With the introduction of the new top-level `subagents/` directory in T1/T2/T3, the install section must also instruct users to copy `subagents/*` into the corresponding `~/.claude/subagents/` (global) or `<repo>/.claude/subagents/` (per-project) location.

The two-tier path scheme parallels the existing skills install pattern exactly, which is the SDD's stated `## Updated install layout` directive: a future plugin packaging maps these directories one-to-one without restructuring.

This Task touches README only. It is purely additive in the `## Install` section — no deletions outside that section.

## What Needs to Change

Edit `README.md` `## Install` section only. No edits anywhere else in the file.

- For each of the four install snippets (Option A POSIX, Option A PowerShell, Option B POSIX, Option B PowerShell), add a parallel copy line for `subagents/*` immediately below the existing `skills/*` line. The new lines mirror the existing line's syntax exactly — same flag (`-r` / `-Recurse`), same path style (`~/...` POSIX, `$HOME\...` PowerShell, per-repo placeholders for Option B), same trailing slash / backslash convention.
- Lightly extend the intro prose for each option (currently mentions "skills" only) to acknowledge subagent definitions are also part of the install. Sentence-level edit, not a paragraph rewrite.
- Optionally add a one-sentence orientation note inside `## Install` explaining the purpose of `subagents/` — only if it does not duplicate prose elsewhere; if it would duplicate, drop the orientation note (acceptable to ship without it).

Stay surgical: do NOT touch any heading or content outside `## Install`.

## Why

Epic A acceptance criterion #3 requires Option A and Option B to both show the extended copy commands in both POSIX and PowerShell variants. Without these edits, users following the install instructions get a broken setup — `Agent(subagent_type="engineer", ...)` invocations fail because the subagent files were never copied to `~/.claude/subagents/`.
