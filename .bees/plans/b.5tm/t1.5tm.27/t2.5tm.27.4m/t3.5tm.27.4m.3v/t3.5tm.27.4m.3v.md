## Context

The Doc Reviewer subagent wraps the existing `/bees-doc-review` skill in the same thin wrapper pattern. Same fresh-eyes / always-cold rule per the PRD. **Model nuance:** unlike Code Reviewer and Test Reviewer (always Opus), the Doc Reviewer's runtime model is user-chosen (Opus or Sonnet) at the start of a `/bees-execute` or `/bees-fix-issue` run per CLAUDE.md `## Model assignment in execution skills`. The file ships with `model: opus` as the shipped default; downstream Epics B and C handle the runtime override mechanism — exactly the same pattern documented for `doc-writer.md` (T1.S3) and `pm.md` (T3.S1).
