## Context

This Epic's rewrite (sibling Subtasks under Task 2) replaces Agent-Teams orchestration with ephemeral-Agent dispatch against the seven custom subagent types installed at `~/.claude/agents/` by Epic A. Claude Code loads custom subagents only at session start; mid-session installs require a Claude Code restart (or `/agents` to hot-reload). Without an explicit precondition gate, a fresh-install user who runs `/bees-breakdown-epic` before restarting will hit a confusing `Agent type 'X' not found`-style error mid-dispatch with no actionable guidance.

The canonical-pattern reference is `skills/bees-execute/SKILL.md` lines ~25-53 (`## Preconditions` section, post-Epic-B rewrite). That skill ships both an upfront fast-fail (`claude agents` enumeration) and a procedural gate (hard-fail on first dispatch's not-found error). Both must appear here for uniformity across the three execution skills.

The Epic body's `## Precondition reminder` paragraph predates Epic A's post-restart probe — it references the stale path `~/.claude/subagents/`. The verified-correct canonical path (per Epic A's Probe results) is `~/.claude/agents/` (global) and `<repo>/.claude/agents/` (per-project). This Subtask must use the corrected path, not the path from the Epic body.

## What Needs to Change

Add a new top-level `## Preconditions` section in `skills/bees-breakdown-epic/SKILL.md` immediately before the existing `### 0. Choose agent model preference` heading (currently the first content section after the file's `# Epic to Tasks` H1 and `## Workflow` H2). The new section's content must mirror `bees-execute/SKILL.md`'s Preconditions section (lines ~25-53) at this skill's smaller scope:

- Hard-fail bullet for the seven required custom subagent types: `engineer`, `test-writer`, `doc-writer`, `pm`, `code-reviewer`, `test-reviewer`, `doc-reviewer`. Even though breakdown only dispatches engineer / doc-writer / test-writer / pm in research mode, the precondition is uniform across the three execution skills — a downstream user repo with tests or reviewer-driven workflows may also dispatch the others, and a uniform check catches "user forgot to restart Claude Code" identically for all three skills.
- Hard-fail bullet for the Plans hive being colonized (`bees list-hives` returns a hive whose `normalized_name` is `plans`).
- Hard-fail bullet for CLAUDE.md `## Documentation Locations` section presence.
- Hard-fail bullet for CLAUDE.md `## Build Commands` section presence with all five required keys (`Compile/type-check`, `Format`, `Lint`, `Narrow test`, `Full test`).
- "Verifying the subagents precondition" subsection with the same hybrid (procedural gate + upfront `claude agents` fast-fail) shape as bees-execute. Include the paired POSIX bash + Windows PowerShell `claude agents` snippets.
- Hard-fail message must direct user to (a) verify install per `README.md` `## Install` AND (b) restart Claude Code or run `/agents` to hot-reload — phrased identically to bees-execute's hard-fail message.
- Cite Claude Code's session-load semantics (subagents loaded at session start; mid-session installs require restart or `/agents` hot-reload).
- All path references use `~/.claude/agents/` (global) and `<repo>/.claude/agents/` (per-project). NOT `subagents/`.

The new Preconditions section is the entire scope of Task 1. Section 0 onward is preserved verbatim by Task 2's preservation Subtasks.
