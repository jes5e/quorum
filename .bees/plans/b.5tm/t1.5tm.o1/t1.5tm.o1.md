## What this Epic delivers

Rewrite `skills/bees-breakdown-epic/SKILL.md` using the reconciliation-loop + ephemeral-Agent pattern. The breakdown skill operates in research-only mode: workers research and report Task/Subtask body content; the orchestrator persists tickets via `bees create-ticket`.

## Acceptance criteria

1. `bees-breakdown-epic/SKILL.md` no longer contains any of: `TeamCreate`, `TeamDelete`, `Agent Teams`, `mode: "plan"`, `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS`.
2. Skill prose references the existing subagent types from Epic A but passes a "research mode" instruction in the Agent prompt — workers report findings, do not mutate state. The orchestrator persists Task/Subtask tickets via `bees create-ticket`.
3. Spec-traceability verification preserved: PM subagent (in research mode) reads PRD/SDD and verifies each proposed Task covers a requirement. Output is consumed by orchestrator before tickets are created.
4. Mandatory Task/Subtask description template applied (current behavior preserved).
5. **Demo:** `/bees-breakdown-epic <epic-id>` produces same-quality Tasks/Subtasks as the current Agent Teams version, verified by spot-checking the output structure against the existing template.

## Codebase-green check

`bees-breakdown-epic` runs end-to-end. Output Tasks/Subtasks pass the existing template/quality checks. Other skills unaffected.

## Implementation notes

- **Research mode signaled via prompt, not separate subagent type.** Engineer subagent prompted with "Research mode — do NOT edit files; report only" still has its full tool allowlist but is instructed not to use mutation tools. Trust the prompt. (If the prompt-only approach proves brittle in practice, the fallback is to define separate research-mode subagent types — `engineer-researcher.md` etc. — but that's out of scope here and would be a follow-up issue.)
- **Orchestrator persists tickets.** Workers return JSON-structured findings; orchestrator validates and runs `bees create-ticket` with the appropriate body files. This keeps `bees create-ticket` invocations centralized and avoids worker-side mutation surprises.
- **Sibling-skill consistency check.** After this Epic ships, all three execution skills are off Agent Teams and Epic E can proceed.
