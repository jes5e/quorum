## Context

The current Section 4 ("Create Task Team to Break Task into Subtasks", lines ~145-265) is built on Agent Teams primitives — "Spawn one persistent team", `mode: "plan"`, "team-lead" / "team" vocabulary, inline role instructions for Engineer / Test Writer / Doc Writer / Product Manager, and a `## Team Composition` subsection. Per Epic AC#1, all of this must be removed.

The replacement is the reconciliation-loop + ephemeral-Agent dispatch pattern that landed in `skills/bees-execute/SKILL.md` Section 3 (post-Epic-B rewrite). bees-breakdown-epic's twist is that workers operate in **research-only mode** — they read code, write nothing, return JSON-structured findings as text. The orchestrator (the running Claude Code session) is the only thing that runs `bees create-ticket`. This is a smaller-scope mirror of the canonical pattern; vocabulary and dispatch shape must match bees-execute so a reader of one skill can navigate the others.

This Subtask owns the substrate switch in Section 4. It does NOT reorder Section 5 (that is Subtask 2.2) and does NOT add the anti-patterns / hub-and-spoke / recursive-delegation callouts (that is Subtask 2.3).

## What Needs to Change

In `skills/bees-breakdown-epic/SKILL.md`, replace the contents of Section 4 (currently `### 4. Create Task Team to Break Task into Subtasks`, including its `#### Team`, `#### Team Composition`, and `#### Task Loop` subsections) with new prose covering:

- **Reconciliation-loop tick** — three phases (read state → dispatch fresh research Agents → yield), event-driven (no clock primitives), driven by Agent completion notifications. Mirror the structure of bees-execute Section 3's `#### Reconciliation loop` subsection at this skill's smaller research-only scope.
- **Per-Subtask cold dispatch shape** — `Agent(subagent_type=<role>, run_in_background=true, prompt=<research-mode preamble + ticket body verbatim>)`. Reference the existing subagent types by literal name: `subagent_type: "engineer"`, `subagent_type: "doc-writer"`, `subagent_type: "test-writer"`, `subagent_type: "pm"`. Link each role name to its contract file (`agents/<role>.md`) for the role's responsibilities, tool allowlist, and instructions.
- **Research-mode dispatch-prompt preamble** — explicit instructions to the dispatched subagent that it is operating in read-only research mode: do NOT modify files, do NOT mutate bees state (no `bees create-ticket` / `update-ticket` / `delete-ticket`), return JSON-structured findings as text. The Epic body's Implementation note explicitly states this is signaled via prompt rather than by introducing separate research-mode subagent types — trust the prompt, do not introduce `engineer-researcher` etc.
- **TaskList naming convention** — deterministic, scope-scoped names per dispatched Agent (`engineer-research-<task-scope-id>`, `doc-writer-research-<task-scope-id>`, `test-writer-research-<task-scope-id>`, `pm-research-<epic-id>`). Mirror bees-execute Section 3's `##### TaskList naming convention` shape — same vocabulary, scope-suffix discipline, deterministic-uniqueness rationale.
- **Procedural-gate fallback callback** — if a research dispatch hits an `Agent type 'X' not found`-style error, STOP and emit the hard-fail per the new `## Preconditions` section (Task 1). This is the cross-Task contract from Task 1 to Task 2.
- **Dispatch prompt: quote the ticket body verbatim** — same shape as bees-execute Section 3's `##### Dispatch prompt: quote the ticket body verbatim` subsection, scoped to the research-mode dispatch.

The replacement preserves:

- The "you (the team-lead) coordinate but the Product Manager has final authority on quality and completeness" framing.
- The "carrying forward architectural decisions: reference any caller-provided architectural decisions or constraints in every affected subtask description; use the caller's exact specification" instruction (currently lines ~152-154).

The replacement removes:

- All `mode: "plan"` references.
- All "Spawn one persistent team" prose.
- The inline `##### Team Composition` block listing inline Engineer / Test Writer / Doc Writer / Product Manager Instructions (lines ~201-265). Role contracts now live in `agents/<role>.md` and are referenced by name.
- The `#### Task Loop` "Spawn one persistent team to handle all Tasks" prose (lines ~287-289).
