## Context

Current Section 5 (`### 5. Review Epic`, lines ~291-346) runs in this order:

1. Review quality of Task and Subtasks (defers to Product Manager).
2. After Epic review is complete, **run `bees create-ticket`** for all Tasks/Subtasks.
3. Set Epic / Tasks / Subtasks to `ready` status.
4. **Spec Traceability Review** as a separate subsection that runs after tickets are created.

This ordering is wrong for the new architecture. Under research-only orchestration, the PM is dispatched as a fresh ephemeral Agent in research mode whose findings the orchestrator consumes — and the PM's spec-traceability gap-fill loop must run BEFORE ticket creation, not after, because the loop may dispatch additional research Agents to fill gaps which then change the proposed Subtask set. Creating tickets first and then traceability-reviewing them creates a delete-and-recreate churn surface.

The new ordering:

1. PM Agent dispatched in research mode (per Subtask 2.1's dispatch shape) with `<scoped-marker-resolver-path>` placeholder substituted to the resolved helper path.
2. PM's traceability-table output consumed by the orchestrator.
3. If any GAP rows are present, dispatch additional research Agents (Engineer / Test Writer / Doc Writer in research mode) to author the missing Subtask bodies; loop back to step 2.
4. Only after PM signs off (all rows OK) does the orchestrator run `bees create-ticket`.
5. After ticket creation, set Epic + Tasks + Subtasks to `ready` status.

The PM dispatch prompt must include a `<scoped-marker-resolver-path>` placeholder. **Resolution is special here**: the helper at `scripts/scoped_marker_resolver.py` is in THIS skill's own `scripts/` directory (NOT sibling-resolved from another skill — bees-breakdown-epic IS the skill that ships the helper). Resolve as `<this skill's base directory>/scripts/scoped_marker_resolver.py`. This differs from bees-execute / bees-fix-issue's sibling-resolution pattern.

This Subtask owns the Section 5 reorder. It does NOT touch the Section 6 (Commit New Ticket Files) or Section 7 (Offer Next Steps) prose, both of which are preserved verbatim by Subtask 2.4.
