## Context

The current `skills/bees-fix-issue/SKILL.md` Preconditions section (lines 15-38 pre-rewrite) hard-fails when `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS != "1"` and invokes the bundled `check_agent_teams.py` helper to verify. The Ephemeral-Agent Orchestration Bee removes Agent Teams entirely; this section must instead hard-fail when the seven custom subagent types are not registered in the running Claude Code session — the same precondition pattern Epic 8s (`t1.5tm.8s` / `bees-execute` rewrite) already shipped.

## What Needs to Change

- Remove the `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` precondition bullet.
- Remove the "Verifying the Agent Teams precondition" subsection that invokes `check_agent_teams.py`.
- Add a new precondition bullet requiring the seven subagent types (`engineer`, `test-writer`, `doc-writer`, `pm`, `code-reviewer`, `test-reviewer`, `doc-reviewer`) to be registered in the running Claude Code session.
- Use the same hard-fail vocabulary as the rewritten `bees-execute/SKILL.md` (point users to `README.md '## Install'` AND a session restart / `/agents` hot-reload).
- Add the same hybrid verification mechanism: an upfront fast-fail via `claude agents` enumeration, plus a procedural gate at first dispatch (`Agent type '<name>' not found`-style error). No fallback to `general-purpose`, no skipping the dispatch, no improvising substitutes.
- Provide POSIX bash and Windows PowerShell snippets for the upfront `claude agents` invocation.
- Preserve the other three preconditions verbatim: Issues hive colonized for this repo (`bees list-hives` shows a hive whose `normalized_name` is `issues`), CLAUDE.md `## Documentation Locations` section present, CLAUDE.md `## Build Commands` section present with all five required keys (`Compile/type-check`, `Format`, `Lint`, `Narrow test`, `Full test`).
- Preserve the rationale paragraph explaining that CLAUDE.md is the contract source for project-specific commands and doc paths.

## Why

Without the substrate switch's precondition update, the rewritten skill would either silently fall back to `general-purpose` (masking the install gap that Epic A's probes documented) or hard-fail with a stale error message referring to Agent Teams that no longer exist.

## Cross-Task contract (consumed by sibling Tasks)

The seven subagent type names listed in this section are the **canonical contract** that sibling Tasks 2 (Section 3 dispatch sites) and 3 (Section 4 reviewer dispatches) reference. If the Preconditions list and the dispatch sites disagree on a subagent name, the precondition check passes but the dispatch fails — the failure mode the procedural gate exists to catch. Sibling Tasks 2 and 3 must reference the same seven names verbatim.

## Acceptance Criteria

- The skill no longer references: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS`, `check_agent_teams.py`, "Agent Teams" (in the Preconditions section).
- The Preconditions section lists all seven required subagent types literally.
- The hard-fail message vocabulary points the user to (a) `README.md '## Install'` and (b) restart Claude Code or run `/agents` to hot-reload.
- The POSIX bash and Windows PowerShell snippets for the `claude agents` upfront fast-fail are present and use a single literal command per snippet (no compound shapes).
- The other three preconditions (Issues hive, Documentation Locations, Build Commands with five keys) are preserved verbatim.
- The rationale paragraph is preserved.
