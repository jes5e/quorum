## Context

This Subtask handles Task 3 of Epic t1.5tm.fk as a single section-rewrite pass. Section 4 (lines 323-374 pre-rewrite) of `skills/bees-fix-issue/SKILL.md` currently:

- Forms a review Team after the implementation team finishes.
- Conditionally spawns reviewer roles (Code Reviewer if Engineer was used; Test Reviewer if Test Writer was used; Doc Reviewer if Doc Writer was used).
- Carries inline Instructions blocks for Code Reviewer / Test Reviewer / Doc Reviewer (each "Invoke the /bees-<role>-review skill").
- Runs a graduated escalation ladder (~10 min / ~20 min / ~30 min nudges + ps-check verification) for idle teammates.
- Runs an iteration loop: "If feedback requires action, **reform the first team** to do the work" with optional-PM-on-iteration logic.

Epic 8s shipped the canonical pattern in `skills/bees-execute/SKILL.md` Section 5 (Subtask t3.5tm.8s.5b.hy); this Subtask mirrors it at issue scope. Replace team formation with three concurrent `Agent(subagent_type=<reviewer>, run_in_background=true)` invocations against the `code-reviewer`, `test-reviewer`, `doc-reviewer` subagent types Epic A landed in `agents/`. The conditional spawn logic is preserved (only spawn reviewers whose corresponding implementer was dispatched). The iteration loop is preserved with the dispatch shape updated. Optional-PM-on-iteration honors the complex-vs-simple gate from Task 2.

The TaskList naming convention is **issue-scoped** (`code-reviewer-<issue-id>`, `test-reviewer-<issue-id>`, `doc-reviewer-<issue-id>`) — consume Section 3's convention as established in Subtask 2.2. Do NOT introduce a Bee-scoped form; that is `bees-execute`'s convention and is wrong here.

The reviewer role-Instructions blocks (lines 347-368) are owned by `agents/<role>-reviewer.md` per Epic A's lift. Section 4 replaces them with one-line summaries plus pointers to the agent definition files.

The graduated escalation ladder (lines 330-345) is replaced by the reconciliation-loop tick defined in Task 2 — there is no idle teammate to escalate to; cold-dispatched Agents either return or are still running.

The "ignored reviewer feedback MUST be in the per-issue summary" discipline (lines 373-374) is load-bearing and preserved verbatim. The Summary block in Section 7 has a `**Ignored Review Feedback**` field that consumes this list.
