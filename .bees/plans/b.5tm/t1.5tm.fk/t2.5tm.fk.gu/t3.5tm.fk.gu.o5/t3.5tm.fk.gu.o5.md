## Context

This Subtask handles Task 4 of Epic t1.5tm.fk as a single section-rewrite pass. Section 7 (lines 393-417 pre-rewrite) of `skills/bees-fix-issue/SKILL.md` handles per-issue close-out: shutdown agents, mark issue done, format + git commit, `TeamDelete` + `force_clean_team` fallback, output summary.

Epic 8s shipped the canonical commit-procedure pattern in `skills/bees-execute/SKILL.md` Section 4.1 (Subtask t3.5tm.8s.z7.1n) — this Subtask mirrors it at issue scope **with one intentional divergence at the staging-rule layer**: `bees-fix-issue` does NOT stage `.bees/` (Issues live in the `issues` hive and have no in-commit ticket-status accumulation; the issue's status flip from `open` → `done` is captured by the bees CLI metadata transition and does not need a working-tree commit), whereas `bees-execute` DOES stage `.bees/` (Tasks/Subtasks accumulate status changes that must land in the per-Task commit). The per-issue summary block (lines 407-415) is preserved unchanged in shape. The `**Ignored Review Feedback**` field is consumed from Task 3's ignored-feedback discipline.

The batch-vs-single mode flow control (line 417) is preserved unchanged.
