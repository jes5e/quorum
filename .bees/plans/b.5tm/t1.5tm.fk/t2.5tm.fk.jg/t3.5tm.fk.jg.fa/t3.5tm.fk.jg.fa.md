## Context

This Subtask is the first half of Task 2 in Epic t1.5tm.fk. Section 3 of `skills/bees-fix-issue/SKILL.md` (lines 125-321 pre-rewrite) is the orchestration heart — currently ~200 lines of Agent Teams team-formation, message-flow choreography, blocked_on signal handling, pre-dispatch state checks, verbatim-ticket-body rule, and inline role-Instructions blocks for Engineer / Test Writer / Doc Writer / PM. Epic 8s already shipped the canonical pattern in `skills/bees-execute/SKILL.md` Section 3 (Subtask t3.5tm.8s.mp.4k); this Subtask mirrors it at issue scope.

Carrying-forward architectural decisions from the canonical pattern:

- **Per-issue cold dispatch** for Engineer / Test Writer / Doc Writer / PM. Each role spawns a fresh `Agent(subagent_type=<role>, run_in_background=true)`. NO `Agent(name=...)` warming, NO `SendMessage` between roles. PM is dispatched only for complex fixes (orchestrator-direct gate preserved).
- **Reconciliation-loop driven by Agent completion notifications.** Explicit "Anti-pattern: no clock primitives" callout enumerating `/loop`, `ScheduleWakeup`, `CronCreate`, polling.
- **Flat orchestration.** No recursive delegation per official sub-agents docs ("Subagents cannot spawn other subagents").
- **Hub-and-spoke preserved.** Workers do not communicate; the diff is the handoff. No `metadata.blocked_on`, no graduated-escalation ladder.
- **Subagent types referenced by name.** No inline role-prose lifted into SKILL.md — Epic A already moved those into `agents/<role>.md`.

Note the **scope difference vs `bees-execute`**: bees-fix-issue has no Subtask breakdown — there is one implementation pass per Issue. The dispatch scope is therefore **issue scope** (`<role>-<issue-id>`), not Subtask scope. The PM dispatch is also **issue scope** (`pm-<issue-id>`), not Task scope. Do NOT copy `bees-execute`'s `<role>-<task-id>-subtask-<id>` form — that is wrong for this skill.

The "Quote the issue body verbatim" rule from current lines 199-207 is **load-bearing and preserved**, but its framing changes: it now governs the orchestrator's Agent dispatch prompt construction, not a team-lead message-bus dispatch. The rule itself stays untouched; only the framing prose around it changes.

This Subtask establishes the Section 3 skeleton; Subtask 2.2 layers the TaskList progress-UI prose with naming convention and the Scoped-marker PM dispatch wiring on top.
