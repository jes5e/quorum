## What this Epic delivers

Remove Agent Teams setup ceremony from `bees-setup/SKILL.md` and delete the now-unused helper scripts.

## Acceptance criteria

1. `bees-setup/SKILL.md` no longer contains: Agent Teams precondition step, `teammateMode` configuration step, iTerm2 hard-prompt workaround prose, references to `force_clean_team.py` or `check_agent_teams.py`.
2. The two helper scripts no longer exist on disk:
   - `skills/bees-execute/scripts/force_clean_team.py` deleted
   - `skills/bees-execute/scripts/check_agent_teams.py` deleted
3. `/bees-setup` runs end-to-end on a fresh repo: hives colonized, CLAUDE.md `## Documentation Locations` and `## Build Commands` sections written, optional PRD/SDD bootstrap. No Agent Teams or display backend prompts. No prompts about `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS`.
4. The fast-path detection (`detect_fast_path.py`) still works on multi-machine clones — it's not Agent Teams-related.
5. CLAUDE.md retains both required contract sections (`## Documentation Locations`, `## Build Commands`); the Agent Teams precondition is removed from skills' precondition checks.

## Codebase-green check

`/bees-setup` runs cleanly on a fresh repo. All execution skills (post-B/C/D) run without Agent Teams. `/bees-status`, `/bees-plan`, `/bees-plan-from-specs`, `/bees-file-issue`, the three review skills unaffected.

## Implementation notes

- **Sequencing-critical.** This Epic must NOT ship before B, C, and D — removing the Agent Teams precondition before all three execution skills are rewritten leaves users in a state where `bees-setup` says "no special setup needed" but `bees-execute` (or one of its siblings) still hard-fails for lack of Teams. The dependency wiring on this Epic enforces this.
- **Pre-deletion grep audit.** Before deleting the two scripts, grep across the repo for any remaining references: `grep -r "force_clean_team\|check_agent_teams" skills/ docs/ README.md CONTRIBUTING.md CLAUDE.md`. The grep should return zero hits before deletion (post-B/C/D world).
- **`bees-fix-issue`'s sibling-resolved reference to `check_agent_teams.py`** (current `bees-fix-issue/SKILL.md` resolves the script as a sibling of `bees-execute/scripts/`) — this reference disappears as part of Epic C, so by the time Epic E runs the script has no callers.
