# Task 2: Update `CONTRIBUTING.md` install loops, rejected-suggestion note, and rewrite the "Status / type renames history" entry

Covers SDD `## Requirements` SR-4.3.

## Context

Second Task of Epic `t1.mak.84`. `CONTRIBUTING.md` carries four distinct contexts that name the old review-skill directories:

1. The POSIX bash install/check symlink loop (lines 13-16 illustrative) — lists the skill directory names alphabetically; users re-run this loop after a rename to refresh their `~/.claude/skills/<name>/` symlinks.
2. The PowerShell install/check symlink loop (lines 20-26 illustrative) — same content, Windows variant.
3. A rejected-suggestion note (line 79 illustrative) about `bees-test-review`'s read-everything alternative — this is editorial prose explaining a design alternative that was considered and rejected.
4. The "Status / type renames history" entry (line 92 illustrative) that previously documented the prior `code-review` → `bees-code-review` rename, currently endorsing dual-mode standalone use as a precedent. **This entry must be REWRITTEN, not appended to**, to record the new decision and reasoning so future contributors do not cite the stale precedent and revert the standalone-removal.

The two install loops are the load-bearing customer-facing path: a user installing the skills runs the loop once at install time and again after any rename to refresh their symlink set. The new alphabetical order of the three new names re-sorts within the loop list — the contract is "the loop installs all skills, in any order; alphabetical ordering is for readability, not behavior" per SDD `## Existing Behavior` final bullet.

The history-entry rewrite is the editorially substantive piece: a fresh-eyes reviewer reading the history section to understand "what is the current state and why" must see the new state's rationale, not the superseded one. Per PRD `## Acceptance Criteria` #10 and SDD SR-4.3 fourth sentence, the rewrite must capture: (a) the misinvocation surface from `/<x>-review` substring matches against generic prompts; (b) the standalone path's value didn't justify it; (c) the rename to role-based names mirrors `agents/<role>.md`. The load-bearing precedent prose — "standalone invocation remains supported via the existing dual-mode prose" — must be replaced, not left in place beside a superseding entry.

**Note on line numbers.** Line numbers cited in SDD SR-4.3 are illustrative; use `grep` to locate references before editing.

## What Needs to Change

In `CONTRIBUTING.md`, perform four context-specific edits:

1. **POSIX bash install loop** (lines 13-16 illustrative): substitute the three old names with the three new names (`bees-code-review` → `bees-engineer-review`, `bees-test-review` → `bees-test-writer-review`, `bees-doc-review` → `bees-doc-writer-review`). Maintain alphabetical ordering — the new names re-sort within the loop list (post-rename alphabetical order: `bees-doc-writer-review`, `bees-engineer-review`, `bees-spec-review`, `bees-test-writer-review`).
2. **PowerShell install loop** (lines 20-26 illustrative): same substitution, Windows variant. Maintain alphabetical ordering.
3. **Rejected-suggestion note** (line 79 illustrative): replace `bees-test-review` with `bees-test-writer-review`. The surrounding editorial prose about the read-everything alternative is unchanged.
4. **"Status / type renames history" entry** (line 92 illustrative) — **REWRITE, do not append**. The rewritten entry must:
   - Replace the load-bearing "standalone invocation remains supported via the existing dual-mode prose" precedent.
   - Capture the new decision and reasoning: misinvocation surface from `/<x>-review` substring matches generic prompts; standalone path's value didn't justify it; renamed to role-based names mirroring `agents/<role>.md` (engineer, test-writer, doc-writer).
   - Stay in the existing history-entry editorial format (date-prefixed bullet, brief decision-and-reasoning prose) so the section's voice is consistent.

The rewrite is the primary editorial deliverable of this Task; treat it as a thoughtful prose update, not a textual substitution.
