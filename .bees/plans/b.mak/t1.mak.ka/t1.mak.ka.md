# Epic 4: Open-ticket sweep + final verification

Covers SDD `## Requirements` SR-5 (open-ticket sweep) and SR-6 (verification).

## Scope

1. Run `bees execute-freeform-query` against the Plans and Issues hives filtered to open/in-progress/drafted/ready statuses with bodies containing any of the three old skill names:

   ```yaml
   stages:
     - [hive=plans, status_in=[drafted,ready,in_progress]]
     - [hive=issues, status=open]
   filter:
     body_contains_any: [bees-code-review, bees-test-review, bees-doc-review]
   report: [ticket_id, hive, ticket_status, title]
   ```

   (Adjust the filter shape to match the bees CLI's actual filter vocabulary at execution time. The expected post-Epic-1-2-3 result is only `b.mak`, `b.mam`, `t1.mam.qr`, and `t1.mam.a2` — this rename's own Plan Bee, Spec Bee, PRD child, and SDD child, all of which correctly describe the rename and should not be modified.)

2. For any open/in-progress ticket the query surfaces beyond those four self-references: update the body in place via `bees update-ticket --body-file <path>` with the new names substituted. Done/closed tickets are not touched.

3. Run final repo-wide verification:
   - `python -m pyflakes skills/*/scripts/*.py` (the repo's `Lint` build command) — expected to pass without errors.
   - Repo-wide `grep -rE "bees-code-review|bees-test-review|bees-doc-review" .` — expected to return matches only inside (a) the rewritten `CONTRIBUTING.md` "Status / type renames history" entry, (b) closed/done bees ticket files under `.bees/`, and (c) this rename's own self-referential bees tickets (`b.mak`, `b.mam`, `t1.mam.qr`, `t1.mam.a2`).

## Acceptance criteria

- The `bees execute-freeform-query` open-ticket sweep returns only the four self-reference tickets (`b.mak`, `b.mam`, `t1.mam.qr`, `t1.mam.a2`).
- `python -m pyflakes skills/*/scripts/*.py` returns no errors.
- Repo-wide `grep` returns no old-name matches outside the documented exceptions above.

## Dependencies

Up-blocks: Epic 1, Epic 2, Epic 3 (all source-file rename work must be complete for the verification to be meaningful).

## Commit boundary
