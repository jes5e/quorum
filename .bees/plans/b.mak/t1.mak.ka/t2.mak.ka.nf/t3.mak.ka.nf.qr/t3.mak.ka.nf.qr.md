## What Needs to Change

**Note on filter vocabulary.** The SDD's illustrative recipe used `body_contains_any:` and `status_in:` clauses that the bees CLI does NOT support. The actual filter terms (per `bees execute-freeform-query --help`) are: `type=`, `status=` (exact), `hive=` / `hive~`, `id=`, `parent=`, `title~`, `tag~`, `guid=`. There is no body-content filter and no status-list filter. The sweep therefore runs as: query candidates by hive + status, then resolve bodies via `bees show-ticket --ids <id1> <id2> ...` and substring-match the three old names in-process.

1. **Run the sweep — query candidates.** Run four queries (one per (hive, status) pair). The two literal YAML strings work identically in POSIX bash/zsh and Windows PowerShell single-quoted strings.

   ```bash
   bees execute-freeform-query --query-yaml 'stages:
     - [hive=plans, status=drafted]
   report: [title, ticket_status]'
   ```

   ```bash
   bees execute-freeform-query --query-yaml 'stages:
     - [hive=plans, status=ready]
   report: [title, ticket_status]'
   ```

   ```bash
   bees execute-freeform-query --query-yaml 'stages:
     - [hive=plans, status=in_progress]
   report: [title, ticket_status]'
   ```

   ```bash
   bees execute-freeform-query --query-yaml 'stages:
     - [hive=issues, status=open]
   report: [title, ticket_status]'
   ```

   Union the returned `ticket_id` values across all four queries (deduplicate).

2. **Resolve bodies and substring-match.** Pass the deduplicated ID list to a single batched `bees show-ticket --ids <id1> <id2> ...` call. For each returned ticket, check whether its `body` field contains any of the three literal substrings `bees-code-review`, `bees-test-review`, `bees-doc-review`. Collect the IDs whose bodies do.

3. **Compare against the expected eight.** The expected match set is exactly:

   ```
   b.mam, t1.mam.qr, t1.mam.a2,
   b.mak, t1.mak.eo, t1.mak.su, t1.mak.84, t1.mak.ka
   ```

   - If the matched-bodies set equals the eight self-references exactly: the sweep passes with no spillover work. Mark the Subtask `done`.
   - If the matched-bodies set is a strict subset of the eight (some self-reference was inadvertently rewritten by an earlier Epic): surface this as a defect — do NOT silently fix it, halt and report. The eight self-references must continue to carry the old names as part of the rename narrative.
   - If the matched-bodies set contains any ID **outside** the eight: those are spillover targets to update in place. Proceed to step 4.

4. **Update spillover tickets in place (only if step 3 surfaces any).** For each spillover ticket ID:

   a. Fetch the body via `bees show-ticket --ids <id>`.

   b. Apply the three literal-substring substitutions in body text:
      - `bees-code-review` → `quo-engineer-review`
      - `bees-test-review` → `quo-test-writer-review`
      - `bees-doc-review` → `quo-doc-writer-review`

   c. Write the substituted body to a scratch file under `<tempdir>/.quorum/` (per `CLAUDE.md` `## Scratch-file convention`). Create the directory if absent. Pick a collision-resistant filename like `bees-body-<ticket-id>.md`. Do NOT delete the file afterward — the convention is no cleanup on any OS.

      ```bash
      # POSIX (bash / zsh):
      mkdir -p /tmp/.quorum
      ```

      ```powershell
      # Windows (PowerShell):
      New-Item -ItemType Directory -Force -Path "$env:TEMP\.quorum" | Out-Null
      ```

      Use the `Write` tool to author the scratch file (do not echo-redirect via shell).

   d. Update the ticket body via:

      ```bash
      bees update-ticket --ids <ticket-id> --body-file <scratch-path>
      ```

      `--body-file` has no size cap, so this is the correct shape for arbitrary body sizes (no need for `append-ticket-body` here — this is a wholesale rewrite, not an append).

   e. Re-fetch the ticket via `bees show-ticket --ids <id>` and confirm none of the three old substrings remain.

5. **Status discipline.** This Subtask is a `t3` (Subtask) ticket, so the full status ladder applies. Set `status=in_progress` at start and `status=done` at end via `bees update-ticket --ids <subtask-id> --status <value>` (one call per transition, not chained).
