## Context

This Subtask is the final verification gate for Bee `b.mak`'s rename narrative (the eight self-referential rename-narrative tickets under `.bees/plans/b.mak/` and `.bees/specs/b.mam/`). It executes the two verifications that confirm prior Epics (`t1.mak.eo`, `t1.mak.su`, `t1.mak.84`) and Task 1 of this Epic (`t2.mak.ka.nf`, the open-ticket sweep) left the repo in a clean state with respect to the old skill names (`bees-code-review`, `bees-test-review`, `bees-doc-review`).

Covers SDD `## Requirements` SR-6.1 (Lint) and SR-6.2 (repo-wide grep with documented exception classes).

The sibling Task `t2.mak.ka.nf` (open-ticket sweep) runs first within this Epic; this Subtask runs after. Both Lint and grep are short literal commands and form a single cohesive verification step, so they are bundled into one Subtask.

This is verification only — no source-code, doc, or ticket-body edits. If a defect is found, halt and surface for triage rather than auto-fixing; the offending Epic owns its own patch.

## What Needs to Change

1. Run the repo's Lint command exactly as written in `CLAUDE.md` `## Build Commands`:

   ```bash
   # POSIX (bash / zsh):
   python -m pyflakes skills/*/scripts/*.py
   ```

   ```powershell
   # Windows (PowerShell):
   python -m pyflakes skills/*/scripts/*.py
   ```

   Expected: clean exit with no errors.

2. Run the repo-wide grep for the three old skill names. **Either use the first-class `Grep` tool (preferred — it excludes VCS metadata directories like `.git/` by default), or use the literal shell command with an explicit `.git/` exclusion to avoid false-positive pseudo-matches in pack files / loose objects / refs.**

   ```bash
   # POSIX (bash / zsh) — explicit .git/ exclusion to avoid VCS-metadata pseudo-matches:
   grep -rE --exclude-dir=.git "bees-code-review|bees-test-review|bees-doc-review" .
   ```

   ```powershell
   # Windows (PowerShell):
   Select-String -Path (Get-ChildItem -Recurse -File -Exclude '.git').FullName -Pattern "bees-code-review|bees-test-review|bees-doc-review"
   ```

   Classify each match against the three documented exception classes:
   - (a) the rewritten `CONTRIBUTING.md` "Status / type renames history" entry,
   - (b) closed/done bees ticket files under `.bees/` (verify by reading the ticket file's `ticket_status` field — `done` or any closed status),
   - (c) the eight self-referential rename-narrative tickets — explicitly enumerated as: `b.mak`, `t1.mak.eo`, `t1.mak.su`, `t1.mak.84`, `t1.mak.ka` (under `.bees/plans/b.mak/`) plus `b.mam`, `t1.mam.qr`, `t1.mam.a2` (under `.bees/specs/b.mam/`).

   If every match falls inside one of the three exception classes, the verification passes. If any match falls outside, halt — do NOT auto-fix; surface the offending file path(s) and matched line(s) to the orchestrator for triage. The Task is held in `in_progress` until the offending Epic is patched.

3. Report verification outcome in the final assistant message: Lint pass/fail, grep match count, classification breakdown (how many matches per exception class), and any unclassified matches that triggered a halt.
