## Context

First phase of Epic `t1.mak.eo` — rename three skill directories so they align with their respective writer-role counterparts (`engineer`, `test-writer`, `doc-writer`) under `agents/`. Per Epic SDD `## Codebase exploration findings`, each of the three directories contains only `SKILL.md` with no `scripts/` subdirectory, so each rename is a single-file move.

This Subtask is structural-only: it renames directories via `git mv` so git records the moves as renames (preserving history), but does NOT edit any file content. All `SKILL.md` content cleanup — frontmatter `name:` / `description:` / `argument-hint:` updates, removal of "When invoked standalone" branches, and internal cross-references between the three SKILL.md files — is scoped to Task 2 of this Epic and depends on this Subtask completing first.

Preconditions assumed:
- Working tree is on the orchestrator-dispatched branch and the three source directories exist with their current names.
- No other in-flight changes touch `skills/bees-code-review/`, `skills/bees-test-review/`, or `skills/bees-doc-review/`.

Sibling-Epic boundaries (out of scope here):
- Caller updates under `agents/` and execution skills — Epic `t1.mak.su`.
- Top-level doc updates (`README.md`, `CLAUDE.md`, `CONTRIBUTING.md`, `docs/sdd.md`, `docs/prd.md`) — Epic `t1.mak.84`.
- Open-ticket sweep + final verification — Epic `t1.mak.ka`.

## What Needs to Change

Perform three `git mv` directory renames. Per the repo's `CLAUDE.md` `## Bash etiquette in this repo`, each `git mv` MUST be its own Bash tool call — no compound chains (`&&` / `;` / `||`), no pipes, no `$VAR` expansion, no diagnostic tails.

Renames (in any order; they are independent):

1. `skills/bees-code-review/` → `skills/bees-engineer-review/`
2. `skills/bees-test-review/` → `skills/bees-test-writer-review/`
3. `skills/bees-doc-review/` → `skills/bees-doc-writer-review/`

Each as a separate Bash call:

```bash
# POSIX (bash / zsh):
git mv skills/bees-code-review skills/bees-engineer-review
```

```bash
# POSIX (bash / zsh):
git mv skills/bees-test-review skills/bees-test-writer-review
```

```bash
# POSIX (bash / zsh):
git mv skills/bees-doc-review skills/bees-doc-writer-review
```

```powershell
# Windows (PowerShell):
git mv skills/bees-code-review skills/bees-engineer-review
```

```powershell
# Windows (PowerShell):
git mv skills/bees-test-review skills/bees-test-writer-review
```

```powershell
# Windows (PowerShell):
git mv skills/bees-doc-review skills/bees-doc-writer-review
```

Do NOT edit any file content in this Subtask. Do NOT update SKILL.md frontmatter, body prose, or cross-references — those are Task 2's scope.

After the three renames, run `git status` (its own Bash call) to confirm all three moves register as renames (`R`) rather than delete + add pairs.

## Key Files

- `skills/bees-test-review/SKILL.md` — moved (unchanged content) to `skills/bees-test-writer-review/SKILL.md`.
- `skills/bees-doc-review/SKILL.md` — moved (unchanged content) to `skills/bees-doc-writer-review/SKILL.md`.

No other files are touched.

## Acceptance Criteria

- The three new directories exist on disk: `skills/bees-engineer-review/`, `skills/bees-test-writer-review/`, `skills/bees-doc-writer-review/`. Each contains exactly one file: `SKILL.md`.
- The three old directories no longer exist: `skills/bees-code-review/`, `skills/bees-test-review/`, `skills/bees-doc-review/`.
- `git status` shows exactly three rename entries (lines beginning with `R` or `R<score>`) corresponding to the three SKILL.md moves — NOT delete + add pairs. (Git detects renames automatically when content is unchanged, which is the case here because no content edits are performed.)
- No file content changes are present in `git diff` for the three moved files — `git diff --stat HEAD` should show zero lines added and zero lines deleted for any SKILL.md path.
- The repo's Lint command (`python -m pyflakes skills/*/scripts/*.py`) still runs cleanly — the three renamed directories contain no `scripts/` subdirectories so the glob is unaffected, but the run confirms nothing structural broke.
