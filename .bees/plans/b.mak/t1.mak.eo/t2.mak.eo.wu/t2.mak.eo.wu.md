# Task 1: Rename three review-skill directories via `git mv`

Covers SDD `## Requirements` SR-1.1 / SR-1.2 / SR-1.3 (directory rename only — SR-1.4 frontmatter `name:` updates and all SKILL.md content cleanup are scoped to this Epic's Task 2).

## Context

First phase of Epic `t1.mak.eo`. This Task renames three skill directories using `git mv` so git records the move as a rename and preserves history. Per SDD `## Codebase exploration findings`, each of the three directories contains only `SKILL.md` with no `scripts/` subdirectory, so the rename is a single-file move per directory.

This Task is a structural change only. No `SKILL.md` content is edited here — frontmatter (`name:`, `description:`, `argument-hint:`), body cleanup ("When invoked standalone" branch removal), and internal cross-references between the three SKILL.md files are all scoped to Task 2 of this Epic.

## What Needs to Change

Rename three directories via `git mv` (one Bash invocation per rename — no compound chains, per `CLAUDE.md` `## Bash etiquette in this repo`):

- `skills/bees-code-review/` → `skills/bees-engineer-review/`
- `skills/bees-test-review/` → `skills/bees-test-writer-review/`
- `skills/bees-doc-review/` → `skills/bees-doc-writer-review/`

## Cross-Task contracts

- **Task 2 of this Epic depends on this Task's completion.** Task 2 edits `SKILL.md` files at the new paths (`skills/bees-engineer-review/SKILL.md`, `skills/bees-test-writer-review/SKILL.md`, `skills/bees-doc-writer-review/SKILL.md`). If Task 2's Subtasks reference the new paths before this Task lands, they will fail. Task 2 carries `up_dependency` on this Task.
- **No content edits.** This Task does NOT modify any file content — only directory names. SKILL.md cleanup (frontmatter + body + internal cross-refs) is Task 2's scope; mixing the two would break the per-Task review boundary.
- **Sibling Epic boundaries.** This Task does NOT touch:
  - `agents/<role>.md` files or the execution skills (`bees-execute`, `bees-fix-issue`) — sibling Epic `t1.mak.su`'s scope.
  - Top-level docs (`README.md`, `CLAUDE.md`, `CONTRIBUTING.md`, `docs/sdd.md`, `docs/prd.md`) — sibling Epic `t1.mak.84`'s scope.
  - Open ticket bodies — sibling Epic `t1.mak.ka`'s scope.
- **Commit boundary.** Per the Epic body's `## Commit boundary`, Epic 1 + Epic 2 land in a single git commit. This Task's renames will not be committed in isolation; they ship together with Epic 2's caller updates per PRD `## Edge Cases and Error Handling` item 1.

## Why

Per PRD `## Functional Requirements` #1 and SDD SR-1.1/1.2/1.3. The role-based directory names mirror `agents/<role>.md` filenames and eliminate the substring-match misinvocation pathway driving fresh-session agents to invoke `/bees-code-review` against generic prompts.

## Success Criteria

- Three new directories exist on disk: `skills/bees-engineer-review/`, `skills/bees-test-writer-review/`, `skills/bees-doc-writer-review/` — each containing the `SKILL.md` previously located at the old path.
- The three old directories no longer exist: `skills/bees-code-review/`, `skills/bees-test-review/`, `skills/bees-doc-review/`.
- `git status` shows the moves as renames (`R`), not as deletes + adds — confirming `git mv` was used and history is preserved.
