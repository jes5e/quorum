# Task 2: Update renamed `SKILL.md` files (frontmatter + body + internal cross-refs)

Covers SDD `## Requirements` SR-1.4, SR-2.1, SR-2.2, SR-2.3, SR-2.4. Pairs with SR-1.1/1.2/1.3 (directory renames in Task 1).

## Context

Second phase of Epic `t1.mak.eo`. After Task 1's `git mv`s, this Task updates the three renamed `SKILL.md` files: rewrites frontmatter (`name:`, `description:`, removes `argument-hint:`), removes the "When invoked standalone" body branch, and updates internal cross-references between the three SKILL.md files.

Per `CLAUDE.md` `## When editing skills`, the `description` field in a skill's frontmatter is what Claude Code uses to decide whether to invoke the skill — the rewrite of the `description` field is the load-bearing change that closes the description-match misinvocation pathway named in the PRD's Problem Statement.

**Note on line numbers.** Line numbers cited in SDD SR-2 are illustrative as of SDD authoring and may have drifted by execution time. Use `grep` to locate references before editing — the `grep` AC checks at the bottom are the authoritative success signal.

## What Needs to Change

For each of the three renamed `SKILL.md` files:

- `skills/bees-engineer-review/SKILL.md`
- `skills/bees-test-writer-review/SKILL.md`
- `skills/bees-doc-writer-review/SKILL.md`

Per-file changes:

1. **`name:` frontmatter field** — update to match the new directory name (`bees-engineer-review`, `bees-test-writer-review`, `bees-doc-writer-review`).
2. **`description:` frontmatter field** — rewrite to lead with the orchestrator scoping pattern, exact form per skill:
   - `bees-engineer-review`: leads with "Review the Engineer's diff during a `/bees-execute` or `/bees-fix-issue` review cycle. Returns a list of improvement work items for the orchestrator."
   - `bees-test-writer-review`: leads with "Review the Test Writer's test code during a `/bees-execute` or `/bees-fix-issue` review cycle. Returns a list of improvement work items for the orchestrator."
   - `bees-doc-writer-review`: leads with "Review the Doc Writer's documentation during a `/bees-execute` or `/bees-fix-issue` review cycle. Returns a list of improvement work items for the orchestrator."

   The substrings `Standalone` and `ad-hoc` MUST NOT appear anywhere in the rewritten descriptions (case-sensitive).
3. **`argument-hint:` frontmatter line** — delete entirely (the skills no longer accept ad-hoc input).
4. **"When invoked standalone" body branch** — delete the paragraph in the Overview section. Preserve the immediately-following "When invoked by `/bees-execute` or `/bees-fix-issue`" paragraph as the sole branch.
5. **Internal cross-reference between the three SKILL.md files** — update to use the new names:
   - `skills/bees-engineer-review/SKILL.md`: replace `/bees-test-review` with `/bees-test-writer-review` (was at line 39 in pre-rename file; locate via `grep -n "/bees-test-review" skills/bees-engineer-review/SKILL.md`).
   - `skills/bees-test-writer-review/SKILL.md`: replace `/bees-code-review` with `/bees-engineer-review` (was at line 32; locate via `grep -n "/bees-code-review" skills/bees-test-writer-review/SKILL.md`).
   - `skills/bees-doc-writer-review/SKILL.md`: replace `/bees-code-review` with `/bees-engineer-review` (was at line 28; locate via `grep -n "/bees-code-review" skills/bees-doc-writer-review/SKILL.md`).
