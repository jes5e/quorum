# Task 3: Update `bees-spec-review/SKILL.md` references to renamed skills

Covers SDD `## Requirements` SR-2.5.

## Context

Third phase of Epic `t1.mak.eo`. The `bees-spec-review` skill is itself NOT being renamed (it is the in-progress fourth review skill, filed under issue `b.uxa`), but its prose references the three renamed skills in the conceptual-analogs paragraph and in three out-of-scope bullets. Without this update, AC#14's repo-wide `grep` for old names would surface matches in `bees-spec-review/SKILL.md` and fail Epic 4's verification.

This Task is independent of Tasks 1 and 2 — `skills/bees-spec-review/` is not renamed, and its `SKILL.md` content edits do not depend on the directory moves Task 1 performs.

**Note on line numbers.** Line numbers cited in SDD SR-2.5 are illustrative as of SDD authoring; use `grep` to locate references at execution time.

## What Needs to Change

In `skills/bees-spec-review/SKILL.md`, replace each occurrence of an old skill name with its new counterpart:

- `bees-code-review` → `bees-engineer-review`
- `bees-test-review` → `bees-test-writer-review`
- `bees-doc-review` → `bees-doc-writer-review`

Locations per SDD SR-2.5 (illustrative line numbers):

- Line 9 — conceptual-analogs paragraph that lists all three together; replace all three names in one paragraph.
- Lines 50, 51, 52 — out-of-scope bullet block, one reference per line; each line names a single old skill that gets substituted with its new name.

Use `grep -nE "bees-code-review|bees-test-review|bees-doc-review" skills/bees-spec-review/SKILL.md` to locate the exact current line set before editing — the AC `grep` check at the bottom of this Task is the authoritative success signal regardless of line drift.

## Cross-Task contracts

- **No dependency on Tasks 1 or 2.** This file is outside the renamed-directory set; its edits are pure textual substitution. Can run in parallel with Tasks 1 and 2 from a file-conflict perspective; sequenced via the Epic's Task ordering for review-flow tidiness.
- **Substitution-only scope.** This Task does NOT touch `bees-spec-review/SKILL.md`'s frontmatter, body structure, or any prose beyond the textual substitution of the three old skill names. The skill itself is in-progress under issue `b.uxa` and any other edits are out of scope.
- **No callers / top-level docs / tickets.** This Task does NOT update callers, top-level docs, or bees ticket bodies — those are scoped to sibling Epics `t1.mak.su`, `t1.mak.84`, and `t1.mak.ka` respectively.

## Why

Per SDD SR-2.5 and PRD `## Acceptance Criteria` #14. AC#14 requires that a repo-wide `grep` for the old names returns no matches outside (a) the rewritten `CONTRIBUTING.md` history entry, (b) closed/done bees tickets, and (c) the eight self-referential rename-narrative tickets. `skills/bees-spec-review/SKILL.md` is none of those exception classes, so it must be updated.

## Success Criteria

- `grep -nE "bees-code-review|bees-test-review|bees-doc-review" skills/bees-spec-review/SKILL.md` returns no matches.
- The conceptual-analogs paragraph and the three out-of-scope bullets remain semantically intact — only the skill names are substituted; surrounding prose is unchanged.
