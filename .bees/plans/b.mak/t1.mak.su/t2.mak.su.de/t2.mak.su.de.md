# Task 2: Update execution-skill orchestrators (`bees-execute`, `bees-fix-issue`) to reference renamed review skills

Covers SDD `## Requirements` SR-3.5, SR-3.6.

## Context

Second Task of Epic `t1.mak.su`. The two execution-skill orchestrators (`skills/bees-execute/SKILL.md` and `skills/bees-fix-issue/SKILL.md`) reference the three old review-skill names in several distinct contexts: `bees-execute`'s PM dispatch line, ignored-feedback report template, and anti-pattern callout (per SDD `## Codebase exploration findings`); `bees-fix-issue`'s anti-pattern callout. The anti-pattern callouts in both files are explicit "do NOT invoke any of the three review skills for a final cross-lane sweep" prose that names each skill alongside its lane scoping ("scoped to source code / user-facing docs / test files"). That callout's lane-scoping aligns precisely with the new role-based names — the rename strengthens the existing prose rather than contradicting it, and the textual substitution is the only edit needed.

**Note on line numbers.** Line numbers cited in SDD SR-3.5 / SR-3.6 are illustrative; use `grep` to locate references before editing.

## What Needs to Change

Update two files under `skills/`. Per-file substitutions:

1. **`skills/bees-execute/SKILL.md`** — replace `bees-code-review` with `bees-engineer-review`, `bees-test-review` with `bees-test-writer-review`, and `bees-doc-review` with `bees-doc-writer-review` everywhere they appear (lines 246 PM dispatch / 357 ignored-feedback report template / 433 + 467 anti-pattern callout illustrative). All three new names appear in this file.
2. **`skills/bees-fix-issue/SKILL.md`** — replace `bees-code-review` with `bees-engineer-review`, `bees-test-review` with `bees-test-writer-review`, and `bees-doc-review` with `bees-doc-writer-review` everywhere they appear (lines 374 + 407 anti-pattern callout illustrative). All three new names appear in this file.

For both files, the substitutions are mechanical textual replacements. The anti-pattern callouts in both files MUST continue to assert that all three review skills should NOT be invoked for cross-lane sweeps; only the names change, the assertion stays.

Do NOT delete or restructure the callout — only swap the skill-name strings inside it.

## Why

Per SDD SR-3.5 / SR-3.6 and PRD `## Functional Requirements` #7 / `## Acceptance Criteria` #7. The PM dispatch line in `bees-execute` runs at execution time and must name the new skills; the anti-pattern callouts in both files name the skills explicitly when telling the orchestrator NOT to invoke them — leaving old names there would cause the warning prose to dangle on non-existent skill IDs.
