## Context

The pre-collected inventory shows that `file_list_resolver.py`, `detect_fast_path.py`, `force_clean_team.py`, and `check_agent_teams.py` do not subprocess to the bees CLI today — none of them imports `subprocess` to invoke `bees`. Task 3 is a verification-and-document Task to confirm this remains true after Task 1 lands the dispatcher and Task 2 migrates the skills. If any helper does shell out to bees, it must be routed through `skills/_shared/scripts/ticket_backend.py` to preserve Epic A's "bees stays the only backend, but only one place knows that" invariant.

Epic A's Scope explicitly names `skills/bees-setup/scripts/detect_fast_path.py` for this verification — include it explicitly in the audit scope.

## What Needs to Change

- Run a verification grep across all helper scripts: search `skills/*/scripts/*.py` (and the new `skills/_shared/scripts/*.py`, excluding `ticket_backend.py` itself which is allowed to call bees) for any pattern that subprocesses to the bees CLI — at minimum `subprocess.*bees`, `os.system.*bees`, and a literal `"bees "` string in argv contexts.
- Explicitly include `skills/bees-setup/scripts/detect_fast_path.py` in the verification scope (Epic Scope names it).
- If the grep is empty (expected outcome), document the verification commands and their results in the commit message and make no file changes.
- If any helper does subprocess to bees, refactor it to call `ticket_backend.py` instead — either by importing the dispatcher as a Python module (if the helper runs in the same Python interpreter) or by subprocessing the dispatcher script using the verb names, flag names, and JSON output shape documented in `ticket_backend.py`'s module docstring as the single source of truth.
- Confirm the existing helpers' contracts are unchanged: `file_list_resolver.py`'s egg-resolver input/output shape, `detect_fast_path.py`'s scope-glob-to-regex translator, `force_clean_team.py`'s team-clean exit codes, and `check_agent_teams.py`'s precondition exit codes.

## Key Files

- `skills/bees-setup/scripts/file_list_resolver.py` — verify, refactor only if it subprocesses bees.
- `skills/bees-setup/scripts/detect_fast_path.py` — verify (Epic Scope explicitly names this), refactor only if it subprocesses bees.
- `skills/bees-execute/scripts/force_clean_team.py` — verify, refactor only if it subprocesses bees.
- `skills/bees-execute/scripts/check_agent_teams.py` — verify, refactor only if it subprocesses bees.
- (any new helpers landed under Task 1 in `skills/_shared/scripts/` — verify they are limited to `ticket_backend.py` and any internal modules it imports)

## Acceptance Criteria

- A grep for `subprocess`, `os.system`, and the literal token `bees` across `skills/*/scripts/*.py` (excluding `skills/_shared/scripts/ticket_backend.py` and any modules it imports internally) returns zero matches that constitute a CLI invocation. Descriptive comments and docstring references are allowed.
- `skills/bees-setup/scripts/detect_fast_path.py` is explicitly named in the verification output (Epic Scope names it).
- The commit message records the exact grep commands and their outputs so the Epic-closing reviewer can confirm Task 3 in one place.
- No helper script's external contract (CLI argv shape, exit codes, JSON output shape) changes as a result of this subtask.
- If any refactor was required, `python -m pyflakes` on the affected file exits 0, and the helper still passes its existing smoke usage in `bees-setup` / `bees-execute` / `bees-fix-issue`.
