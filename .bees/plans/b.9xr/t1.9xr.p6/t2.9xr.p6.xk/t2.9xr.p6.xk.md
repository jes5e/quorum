# Task 4: Add "Backend dispatcher" subsection and review-criteria bullet to CLAUDE.md

## Context

Task 1 introduces a new design rule: skill prose and helper scripts must call the ticket backend through `skills/_shared/scripts/ticket_backend.py`, not the bees CLI directly. This Task documents that rule in CLAUDE.md (project root) so future contributors and the in-repo review skills (`bees-code-review`, `bees-doc-review`) have an explicit hook to flag direct `bees ...` calls.

The caller has decided this rule lives in CLAUDE.md (relaxes Epic A AC #5's "outside `skills/` and `docs/`"). The new subsection is placed between `## Bash etiquette in this repo` and `## Review criteria for skill changes` so it sits next to the related Bash-etiquette and ticket-querying material.

## Cross-Task contract dependency

The verb names listed in the new subsection match the contract documented in `ticket_backend.py`'s module docstring. CLAUDE.md does NOT restate verb argv shapes or JSON output schemas — those live in the docstring; CLAUDE.md points readers at it as the authoritative source.

## What Needs to Change

- Add a new `## Backend dispatcher` subsection to project-root CLAUDE.md, placed between `## Bash etiquette in this repo` and `## Review criteria for skill changes`.
- Extend the existing `## Review criteria for skill changes` bullet list with a fourth criterion covering direct `bees <subcommand>` invocations in SKILL.md prose or helper scripts.

## Why

A reviewable, durable design rule. Without explicit CLAUDE.md text, the new dispatcher pattern is informal and prone to regression — contributors editing skills six months from now will reach for the old `bees ...` shape if no rule names it as forbidden. The review-criteria bullet makes the rule mechanically reviewable by `bees-code-review` and `bees-doc-review`.

## Success Criteria

- CLAUDE.md has a new top-level `## Backend dispatcher` subsection placed between `## Bash etiquette in this repo` and `## Review criteria for skill changes`.
- The subsection lists the seven verbs by name, defers arg/JSON shapes to the dispatcher's module docstring, forbids direct `bees ...` calls in SKILL.md prose AND helper scripts, and includes a forward note about Epic B's beads branch.
- `## Review criteria for skill changes` lists four bullets total — the original three plus a new dispatcher-rule bullet styled to match.
- The additive-criteria framing and project-scoped caveat in `## Review criteria for skill changes` are preserved verbatim.
