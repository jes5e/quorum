# Epic A — Introduce ticket-backend dispatcher (refactor)

**Status at completion.** Codebase green; bees remains the only supported backend; no user-visible behavior change.

## Description

Create the new shared helper directory `skills/_shared/scripts/` and add `ticket_backend.py` — a single dispatcher exposing the verb-shaped subcommands the SDD specifies (`query`, `create`, `update`, `show`, `list-spaces`, `setup-spaces`, `resolve-spec`). Internally, the dispatcher reads the backend choice from CLAUDE.md `## Ticket Backend` (defaulting to `bees` when the section is absent, so this Epic is purely additive), and shells out to the bees CLI behind each verb. All eleven portable-core skills' `SKILL.md` files get every `bees <subcommand>` invocation in their prose replaced with a single `python3 "<base-dir>/../_shared/scripts/ticket_backend.py" <verb> ...` call. Both POSIX and PowerShell snippet pairs collapse into one Python invocation each, since Python is already cross-platform. CLAUDE.md gets a new "Backend dispatcher" subsection in the design-rules section explaining the new pattern and the prohibition on direct `bees ...` calls in skill prose.

This is a pure-tech refactor — the SDD's "Sequencing" paragraph explicitly allows this exception so that Epic B can be purely additive. Every existing query/mutation pattern catalogued in the freeform-query inventory must still work end-to-end after this Epic.

## Scope

- Create `skills/_shared/scripts/ticket_backend.py` and (if Python module structure helps) an empty `skills/_shared/scripts/__init__.py`.
- Implement all seven verbs (`query`, `create`, `update`, `show`, `list-spaces`, `setup-spaces`, `resolve-spec`) with bees-only routing.
- Define and document the JSON output shape per verb in the dispatcher's docstring.
- Update all 11 skills (`bees-setup`, `bees-plan`, `bees-plan-from-specs`, `bees-breakdown-epic`, `bees-execute`, `bees-fix-issue`, `bees-file-issue`, `bees-status`, `bees-code-review`, `bees-doc-review`, `bees-test-review`) to call the dispatcher instead of `bees ...` directly.
- Verify and revise `bees-setup`'s `detect_fast_path.py` if it shells out to `bees` directly.
- Add a new "Backend dispatcher" subsection to CLAUDE.md's design rules.
- Run `python -m pyflakes skills/*/scripts/*.py skills/_shared/scripts/*.py` — zero new lint errors.

## Acceptance criteria

1. **Demo: `/bees-status` parity.** Agent runs `/bees-status` on this very repo before and after the refactor and the human-readable output is byte-for-byte identical (modulo any timestamps).
2. **Demo: dispatcher coverage.** Agent runs `grep -nE "bees (execute-freeform-query|show-ticket|create-ticket|update-ticket|list-hives|colonize-hive|set-types|set-status-values)" skills/*/SKILL.md` and the result is empty — no skill prose calls bees directly anymore. (Calls inside `_shared/scripts/ticket_backend.py` are the dispatcher's territory and are excluded from this check.)
3. **Demo: end-to-end smoke.** Agent runs `/bees-plan` on a one-line scope ("Add a hello-world function"), confirms a Plan Bee gets created with one Epic, and the resulting `bees show-ticket` output matches the same shape as before the refactor.
4. **Demo: lint clean.** `python -m pyflakes skills/_shared/scripts/*.py` exits 0.
5. **Codebase green.** No file outside `skills/` and `docs/` is touched. The `b.veq` open issue (review skills work) is unaffected.

- The `## Ticket Backend` CLAUDE.md contract section (deferred to Epic B — Epic A defaults to `bees` when missing).
