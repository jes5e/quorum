# Task 1: Create the ticket-backend dispatcher with seven verbs (bees-only routing)

## Context

Epic A introduces `skills/_shared/scripts/ticket_backend.py` — a single dispatcher between skill prose and a ticket-backend CLI. This Task lays down the dispatcher itself: the directory, the module, all seven verbs (`query`, `create`, `update`, `show`, `list-spaces`, `setup-spaces`, `resolve-spec`), the `## Ticket Backend` reader (defaults to `bees` when section is absent — Epic A is purely additive), and the JSON output shape per verb documented in the module docstring.

This Task does NOT migrate any skill prose — that is Task 2's job. This Task does NOT add the beads branch — that is sibling Epic B (`t1.9xr.4e`)'s job. The dispatcher's beads branch returns a stable "not yet supported" error in this Task.

## Cross-Task contract (load-bearing for the rest of the Epic)

The verb argv shapes, flag names, and JSON output schemas that this Task pins down in `ticket_backend.py`'s module docstring are a contract that:

- **Task 2** (skill-prose migration) consumes — every migrated invocation must match this contract.
- **Task 4** (CLAUDE.md design-rules subsection) references — the subsection lists verb names and points readers at the docstring as the authoritative source rather than restating verb argv or JSON shapes.
- **Sibling Epic B** (`t1.9xr.4e`)'s beads branch must respect — beads' reads/writes must produce equivalent normalized JSON.

If this Task changes the verb names, flag names, or JSON shapes, every dependent Task is re-affected. The docstring is the single source of truth.

## What Needs to Change

- Create `skills/_shared/scripts/ticket_backend.py` and `skills/_shared/scripts/__init__.py`.
- Implement all seven verbs with bees-only routing.
- Document JSON output shape per verb in the module docstring (the cross-Task contract reference).
- Implement the `## Ticket Backend` reader with the bees default.
- Wire a stable "beads not yet supported" error path returning exit code 2 with a stable message naming the verb and pointing at Epic B.
- Run `python -m pyflakes skills/_shared/scripts/*.py` clean.

## Why

The PRD calls for a backend-pluggable workflow; the SDD splits the work into two Epics so Epic A is a pure refactor with no user-visible behavior change. This Task establishes the seam Epic B will fill.

## Success Criteria

- `skills/_shared/scripts/ticket_backend.py` exists with all 7 verbs and bees-routing.
- The module docstring documents per-verb argv shapes and JSON output schemas, plus a stability/cross-Task contract pointer.
- The `## Ticket Backend` reader defaults to `bees` when the section is absent.
- `python -m pyflakes skills/_shared/scripts/*.py` exits 0 (Epic AC #4).
- A smoke comparison between `python3 ticket_backend.py query --query-yaml '<recipe>'` and `bees execute-freeform-query --query-yaml '<recipe>'` produces byte-identical stdout on this repo.
- A smoke comparison between `python3 ticket_backend.py list-spaces` and `bees list-hives` produces byte-identical stdout on this repo.
- Calling any verb on a CLAUDE.md whose `## Ticket Backend` is set to `beads` exits with code 2 and the stable Epic-B-pointer error message.
