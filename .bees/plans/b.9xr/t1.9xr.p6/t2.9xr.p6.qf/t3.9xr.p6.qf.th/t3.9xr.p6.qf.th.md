## Context

Every verb's first action at runtime is "what backend am I dispatching to?" The dispatcher answers that by reading `## Ticket Backend` from the target repo's CLAUDE.md. Epic A is purely additive — when the section is missing, the reader returns `bees`, so existing repos keep working unchanged. Epic B (sibling) will tighten this to a hard-fail when the section is absent; the reader's signature should leave room for that future change without touching every verb.

## What Needs to Change

Inside `ticket_backend.py`, add a backend-reader helper that:

- Locates `<repo-root>/CLAUDE.md` using the top-level `--repo-root` flag from Subtask 1.1.
- Parses the file's top-level `## Ticket Backend` section using the same fenced-block-aware section parser style as `detect_fast_path.py` (look at its `_split_top_level_sections` / `_parse_keyed_bullets` pattern for precedent — match the style, don't duplicate code).
- Returns the bullet value (`bees` or `beads` — single string) when the section is present and well-formed.
- Returns `"bees"` when the section is absent, the file is missing, or the section is empty. This is the Epic A default.
- Errors with a clear stderr message and non-zero exit code if the section is present but contains an unrecognized backend value (anything other than `bees` or `beads`). Listing `beads` here is fine even though the beads branch lands in Epic B — the reader recognizes the value, and the verb-level dispatch raises a "not yet supported" error in Subtask 1.5.
- Returns its result via a small named tuple or dataclass-shaped dict so future Epic B logic can attach extra context (per-database paths, etc.) without changing the reader's call sites.

Wire the reader into the dispatcher's `main()` so every verb resolves the backend before invoking its handler. Verbs receive the resolved backend as an argument; do not have each verb re-read CLAUDE.md.

## Key Files

- `skills/_shared/scripts/ticket_backend.py` — add backend-reader helper and wire it into `main()`.

## Acceptance Criteria

- Calling any verb on a repo whose CLAUDE.md lacks `## Ticket Backend` resolves to `bees` and proceeds.
- Calling any verb on a repo whose CLAUDE.md has `## Ticket Backend` set to `bees` resolves to `bees` and proceeds.
- Calling any verb on a repo whose CLAUDE.md has `## Ticket Backend` set to `beads` resolves to `beads` (the verb-level handler then errors per Subtask 1.5; that error is part of the next subtask, not this one).
- Calling any verb on a repo whose CLAUDE.md has `## Ticket Backend` set to anything else exits non-zero with a clear stderr message naming the offending value.
- Section parsing is fenced-block-aware (a `## Ticket Backend` literal that appears inside a fenced code block does not satisfy the section check).
- The reader's return value is shaped to allow future Epic B context attachment (named tuple or dataclass-style dict, not a bare string).
- `python -m pyflakes skills/_shared/scripts/ticket_backend.py` exits 0.
