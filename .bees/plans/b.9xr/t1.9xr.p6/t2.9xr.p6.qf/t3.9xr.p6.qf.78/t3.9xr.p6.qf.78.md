## Context

Lays down the dispatcher's foundation: directory, module file, argparse skeleton, and the per-verb JSON output contract documented in the module docstring. The docstring is load-bearing: Task 2 (skill-prose migration) and Task 4 (CLAUDE.md design-rules subsection) both reference it as the single source of truth for verb argv shapes and JSON output schemas. Sibling Subtasks 1.2–1.5 fill in implementation behind the contract this subtask pins down.

## What Needs to Change

- Create `skills/_shared/scripts/` with an empty `__init__.py` (so future tests can import the module).
- Create `skills/_shared/scripts/ticket_backend.py` with:
  - A short module summary (one or two lines, per `docs/doc-writing-guide.md`) plus a comprehensive module docstring documenting per-verb argv shape, JSON output shape on success, JSON output shape on failure, and exit-code semantics for all seven verbs (`query`, `show`, `create`, `update`, `list-spaces`, `setup-spaces`, `resolve-spec`).
  - The argparse skeleton: top-level parser with a `subparsers` group, one subparser per verb, plus a top-level `--repo-root` flag defaulting to the current working directory.
  - A `main()` that dispatches to per-verb handler stubs (`NotImplementedError` for now; later subtasks fill them in).
  - Standard `if __name__ == "__main__": sys.exit(main())` entry point.
- The argparse flag names mirror the corresponding bees CLI flags wherever the verb maps onto a single bees subcommand (`--ids`, `--ticket-type`, `--hive`, `--query-yaml`, `--body-file`, etc.) so skill prose readability is preserved when Task 2 swaps the binary.
- Note in the docstring that "Tasks 2 and 4 of Plan Bee b.9xr Epic A consume this contract — do not change verb names, flag names, or JSON keys without updating those Tasks."

## Key Files

- `skills/_shared/scripts/ticket_backend.py` — new module file with skeleton + docstring.
- `skills/_shared/scripts/__init__.py` — new, empty.

## Acceptance Criteria

- `skills/_shared/scripts/ticket_backend.py` exists, is executable Python 3, and loads without errors when imported.
- The module docstring documents argv shape and JSON output shape for all seven verbs and the exit-code contract.
- The argparse skeleton accepts every verb name (`query`, `show`, `create`, `update`, `list-spaces`, `setup-spaces`, `resolve-spec`) and rejects unknown verbs with a non-zero exit code.
- An empty `__init__.py` exists in `skills/_shared/scripts/`.
- Running each verb without arguments produces the verb's usage message; running it with `--help` works without raising.
- `python -m pyflakes skills/_shared/scripts/ticket_backend.py skills/_shared/scripts/__init__.py` exits 0.
- The argparse flag names mirror the corresponding bees CLI flags wherever the verb maps onto a single bees subcommand.
- The docstring contains the cross-Task contract pointer for Tasks 2 and 4.
