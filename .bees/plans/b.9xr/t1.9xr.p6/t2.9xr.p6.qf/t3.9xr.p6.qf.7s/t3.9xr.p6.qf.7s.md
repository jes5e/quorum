## Context

The three read verbs are pass-through: they translate the dispatcher's argparse arguments into a single bees CLI invocation, run it, propagate stdout/stderr/exit-code, and exit. They are the easiest verbs and the highest-traffic verbs in skill prose, so getting the contract right here unblocks the most Task 2 prose-edit volume.

## What Needs to Change

Implement the bees branches of:

- `query` — accepts `--query-yaml '<yaml literal>'` and shells out to `bees execute-freeform-query --query-yaml '<yaml>'`. The YAML literal is forwarded verbatim — the dispatcher does not parse it. Stdout from bees (the existing JSON `{"status": "...", "tickets": [...]}` shape) passes through unmodified so skill prose can keep using its existing parsing recipes.
- `show` — accepts `--ids <id1> <id2> ...` (variadic) and shells out to `bees show-ticket --ids <id1> <id2> ...`. Stdout pass-through. The existing bees JSON shape (`{"status": "success", "tickets": [...], "not_found": [...], "errors": [...]}`) is the contract.
- `list-spaces` — no arguments; shells out to `bees list-hives`. Stdout pass-through. The existing bees JSON shape (`{"status": "success", "hives": [{"normalized_name": ..., "path": ..., "scope": ...}]}`) is the contract. The dispatcher's verb name is "list-spaces" because beads has no concept of hives, but on bees the JSON keys remain `hives` (Epic B's beads branch will normalize the key naming).

Use `subprocess.run` with `check=False` so a non-zero exit from bees propagates; capture stdout and stderr, write them through unmodified, and exit with the bees CLI's exit code. Do not buffer or reformat — the JSON contract is bees' JSON.

Update the module docstring's per-verb JSON-shape section to reaffirm `list-spaces` returns the bees `hives` key in Epic A — Epic B will rename or alias.

## Acceptance Criteria
