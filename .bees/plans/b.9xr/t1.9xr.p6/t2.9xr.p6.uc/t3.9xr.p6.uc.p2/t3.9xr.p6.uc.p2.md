## Context

The OS-conditional shell-block rule predates the dispatcher and is written assuming most snippets are direct CLI calls (`bees ...`, `which bees`, etc.) that have shell-specific syntax. After Task 2, the bulk of skill-prose snippets are `python3 ".../ticket_backend.py" <verb> ...` invocations — a single literal command that works identically on POSIX bash/zsh and Windows PowerShell. The guide's existing "every shell snippet ships labeled for at least POSIX bash and native Windows PowerShell" rule is preserved in spirit, but the worked-example shape needs an explicit carve-out so future skill authors do not pad dispatcher calls into spurious paired blocks.

## What Needs to Change

- Add a short subsection or paragraph inside `## OS-conditional shell blocks` clarifying that dispatcher invocations (`python3 "<base-dir>/../_shared/scripts/ticket_backend.py" <verb> ...`) ship as a single labeled block covering both shells, since `python3` and the path-quoting form work identically in POSIX bash/zsh and PowerShell.
- Preserve the existing rule that the snippet must still carry an OS comment label so intent is explicit; the change is "one block, dual-OS label" not "no label at all".
- Call out the existing exception that already applies — when a recipe interpolates a shell variable (e.g., a ticket ID) into the dispatcher command, the interpolation syntax is shell-specific and the recipe still needs OS-paired variants. This is the same caveat the `## Querying tickets` cross-platform note already carries; phrase the two consistently.
- Do not weaken the rule for non-dispatcher snippets (`which bees`, `Get-Command`, env-var probes, etc.) — those still require paired POSIX + PowerShell blocks.

## Key Files

- `docs/doc-writing-guide.md` — `## OS-conditional shell blocks` section.

## Acceptance Criteria

- The section explicitly documents that dispatcher invocations are a single labeled block, not a paired pair, with a one-line rationale (Python is cross-platform).
- The OS comment-label requirement is preserved verbatim for dispatcher snippets — only the count of blocks collapses, not the labeling discipline.
- The shell-variable-interpolation exception is stated and worded consistently with the parallel note in `## Querying tickets`.
- Non-dispatcher snippet rules (paired POSIX + PowerShell, no bash-only fallback) are unchanged.
