## Context

Task 2 of Epic A migrates every direct `bees <subcommand>` invocation in skill prose to a single `python3 "<base-dir>/../_shared/scripts/ticket_backend.py" <verb> ...` call. The doc-writing guide is the canonical reference for skill authors and currently prescribes the soon-to-be-wrong direct-shell pattern in its `## Querying tickets` section. The Engineer is rewriting the skill prose itself; this subtask covers the guide that teaches future skill authors which pattern to follow. The dispatcher's `query` verb forwards the YAML literal unchanged, so all existing recipes remain valid in terms of their YAML payload — only the binary changes.

## What Needs to Change

- Replace the "the verb is always `bees execute-freeform-query --query-yaml '<yaml>'`" prescription in `## Querying tickets` with the dispatcher-shaped equivalent calling the `query` verb on `ticket_backend.py`, resolved through the standard skill-relative `<base-dir>/../_shared/scripts/...` path convention already documented in `## The lookup-key pattern`.
- Update every worked-example recipe in the section so each shell block uses the dispatcher invocation. The single-quoted YAML literals stay byte-identical; only the command preamble changes.
- Keep the surrounding prose about the bees CLI not having an `ls` / `search` / `list-tickets` verb, but reframe it so it explains why the dispatcher's `query` verb is the only enumeration entrypoint and direct `bees execute-freeform-query` calls are no longer permitted in skill prose.
- Refresh the cross-platform note at the bottom of the section to reflect that dispatcher calls are a single Python invocation that works on POSIX and PowerShell unchanged (referencing the parallel update in `## OS-conditional shell blocks` per Subtask 2.D2).
- Defer the authoritative verb list and JSON output shapes to the dispatcher's module docstring (the cross-Task contract reference Task 1 establishes); do not re-list verb names or per-verb JSON keys inside this guide section.

## Key Files

- `docs/doc-writing-guide.md` — `## Querying tickets` section.
- `skills/_shared/scripts/ticket_backend.py` — module docstring (read-only reference; do not modify here).
