## Context

`bees-setup/SKILL.md` is the most complex migration target: it contains paired POSIX bash + Windows PowerShell snippet pairs for each hive's `colonize-hive` + `set-types` + `set-status-values` triplet, plus `list-hives` / `get-types` / `get-status-values` verification calls and a `show-ticket` verification step. The dispatcher's `setup-spaces` composite verb (per Task 1's contract) is designed to absorb the per-hive triplet into a single call, collapsing six OS-paired snippet pairs into one Python call per hive.

**Sibling Epic B drift watch.** This subtask's biggest risk is reaching into Epic B's scope. Epic B (sibling, `t1.9xr.4e`) owns: the `## Ticket Backend` writer in `bees-setup`, the backend-pick step at the top of the skill, the beads-conditional branches in Prerequisites and Hive Configuration, and the egg-resolver subsection's bees-only conditional. **None of those are in Epic A's scope.** This subtask is migration-only — the migrated prose still always writes the two existing CLAUDE.md sections (`## Documentation Locations`, `## Build Commands`) and never writes `## Ticket Backend`.

## What Needs to Change

- Replace every direct `bees <subcommand>` call in `bees-setup/SKILL.md` prose with the dispatcher equivalent, using the verb names, flag names, and JSON output shape documented in `ticket_backend.py`'s module docstring as the single source of truth.
- Use the dispatcher's `setup-spaces` composite verb wherever the prose currently issues a `colonize-hive` + (optional) `set-types` + `set-status-values` triplet for a single hive. Pass child-tier and status-value JSON through the dispatcher exactly as the bees CLI received them, preserving the "omit set-types when child tiers are 'none'" branch in prose form.
- Use `list-spaces` and the dispatcher's equivalent of `get-types` / `get-status-values` (or whatever the dispatcher names them per its docstring) for the verification reads.
- Use the dispatcher's `show` verb for the post-setup `show-ticket` verification of a Plan Bee with eggs.
- Collapse paired POSIX bash + PowerShell blocks into a single `python3 ...` invocation per call site, since Python is cross-platform. Where the surrounding prose still needs OS-conditional branches (temp file paths, line-continuation characters in non-dispatcher commands), preserve those — only the dispatcher invocations themselves collapse.
