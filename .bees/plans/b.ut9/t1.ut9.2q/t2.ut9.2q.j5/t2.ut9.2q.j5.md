## Context

The three reviewer-subagent wrapper files — `agents/code-reviewer.md`, `agents/doc-reviewer.md`, `agents/test-reviewer.md` — are thin pass-through wrappers that dispatch their wrapped review skill (`/quo-engineer-review`, `/quo-doc-writer-review`, `/quo-test-writer-review` respectively) and defer to the wrapped skill's output contract. Each wrapper's **Instructions-block output-shape sentence** currently describes that contract as: *"Return findings to the orchestrator as a structured list consistent with the wrapped skill's existing output contract: severity tags (`blocker` / `suggestion` / `nit`), file:line references, suggested fixes, and a verdict."* (line 23 in `code-reviewer.md` and `test-reviewer.md`; line 27 in `doc-reviewer.md`).

This Epic (SR-1.4) aligns that output-shape descriptor to **also** mention the new per-fix-path depth dimension, so the wrapper contract stays literal now that the wrapped skills emit it (Epics `t1.ut9.9c` / `t1.ut9.jn` / `t1.ut9.q2`). The wrappers continue to defer to the wrapped skills' output contracts — this is a pass-through documentation alignment, NOT a behavioral change.

**Spec-wording nuance (resolved via the authoritative SDD).** The Epic body says "frontmatter alignment only" and AC #3 says "no Instructions-block edits", but the existing severity-tag mention being aligned actually lives in each wrapper's **Instructions-block output-shape sentence**, NOT in the YAML frontmatter `description` (the `description` fields do not mention severity tags). SDD `t1.tip.ux §SR-1.4` is authoritative and references this as the "Output-shape contract at lines 22-23" — i.e., the Instructions-block descriptor sentence. Resolution: add the depth-dimension mention to that output-shape descriptor sentence (the one that already names severity tags), alongside the existing severity-tag language. AC #3's "no Instructions-block edits" is read per its evident intent — **do NOT change the pass-through behavior**: the Responsibilities block, the Cold-start invariant, the `description`/`model`/`tools` YAML frontmatter, the "invoke the wrapped skill / do not redefine the output shape / defer to whatever the wrapped skill produces" instructions all stay unchanged. The ONLY edit is adding the depth-dimension phrase to the single output-shape descriptor sentence that already lists severity tags. This does not redefine the output shape (the wrapper still defers to the wrapped skill); it makes the descriptor literal.

## What Needs to Change

In each of `agents/code-reviewer.md`, `agents/doc-reviewer.md`, `agents/test-reviewer.md`:

- Edit the single output-shape descriptor sentence (the one that currently reads "... severity tags (`blocker` / `suggestion` / `nit`), file:line references, suggested fixes, and a verdict ...") to ALSO mention the **per-fix-path depth tags** (`trivial-tweak` / `refactor-locally` / `re-architect`) and the enumerated fix paths, alongside the existing severity-tag mention. Keep the "defer to whatever `/quo-<wrapped-skill>` produces / do not redefine the output shape" clause intact — the wrapper still defers; the descriptor just now names both dimensions.
- Reference the extended emission shape from the respective wrapped skill: `code-reviewer.md` → `/quo-engineer-review` (Epic `t1.ut9.9c`); `doc-reviewer.md` → `/quo-doc-writer-review` (Epic `t1.ut9.jn`); `test-reviewer.md` → `/quo-test-writer-review` (Epic `t1.ut9.q2`).
- Do NOT edit: the YAML frontmatter (`name` / `description` / `model` / `tools`), the Responsibilities block, the Cold-start invariant block, the `doc-reviewer.md` "Model default and runtime override" block, or the pass-through instructions (read scope / invoke wrapped skill / do not compute scope). Only the output-shape descriptor sentence changes.

## Why

The wrapper-side contract describes an aspirational output shape; before the emission Epics, it mentioned severity tags the wrapped skills did not actually emit. Now that Epics `9c` / `jn` / `q2` make the wrapped skills emit severity + per-fix-path depth + enumerated fix paths, this Epic makes the wrapper descriptors literal again by naming the depth dimension. Closes the pre-existing wrapper-vs-wrapped contract drift the SDD calls out.

## Success Criteria

- All three wrapper files' output-shape descriptor sentence mentions severity tags AND per-fix-path depth tags (and the enumerated fix paths) as part of the wrapped-skill output contract.
- A `grep` for the new depth-dimension language across `agents/*-reviewer.md` returns three hits (one per file).
- The wrappers' pass-through-to-wrapped-skill behavior is unchanged: the Responsibilities block, Cold-start invariant, YAML frontmatter, and the "defer to the wrapped skill's output contract" instruction are all untouched.
- The change is pure prose: language-agnostic, project-neutral, no new shell snippets (no OS-pairing obligation), no scratch-file writes.
