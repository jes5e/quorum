Extend `skills/quo-engineer-review/SKILL.md` Step 4 (Generate Work Item List) so each finding emits a severity tag (`blocker` / `suggestion` / `nit`) AND one or more enumerated fix paths, each with its own depth tag (`trivial-tweak` / `refactor-locally` / `re-architect`). This is the *first* introduction of severity tagging to this skill — findings currently carry no severity tag.

## Scope

- Document the new output shape in the SKILL.md's output-shape section (currently at lines 148–175).
- Provide at least one worked example showing severity + multi-path-with-depth + single-path-with-depth.
- Preserve the existing two-trailer-shape pattern (findings-present / clean) — the new emission shape sits *inside* the work-item list; the trailer at the end is unchanged in form.
- Update any examples in the skill prose that show finding shapes to match the new contract.

## Acceptance criteria

1. The skill's output-shape section documents severity tags AND per-fix-path depth tags AND enumerated fix paths.
2. At least one worked example renders in the skill prose with the new shape.
3. The two existing trailer shapes (findings / clean) are preserved verbatim.
4. A live run of `/quo-engineer-review` against any diff produces findings in the documented shape.

Out of scope: changes to other review skills (separate Epics), changes to `agents/code-reviewer.md` (Epic 6), changes to orchestrator routing (Epic 7).
