## Context

`/quo-plan` Step 5e (the fresh-eyes plan-review gate) embeds a read-only reviewer **dispatch prompt** whose output contract is at `skills/quo-plan/SKILL.md` lines ~511–609. This is an **embedded dispatch-prompt output contract**, NOT a standalone review skill's output section (per SDD SR-1.3). The reviewer **already** emits:
- a backtick severity tag — `` `blocker` `` / `` `suggestion` `` / `` `nit` `` (lines 511–513),
- a `target:` tag — exactly one of `PRD` / `SDD` / `Plan-Bee-body` / `Epic:<epic-id>` (lines 514–515),
- a one-or-two-sentence description,
- a verdict trailer line — `Plan-review verdict: <approve|revise-recommended|escalate-to-user>` (lines 532–535),
- a verdict-conditional routing trailer — Shape A / B / C (lines 561–609).

This Epic adds **only** the orthogonal per-fix-path depth dimension and fix-path enumeration to the finding shape. Severity and the `target:` tag are already present and stay. The reviewer's substantive holistic-judgment lane (already correctly framed) is a PRD non-goal — do NOT touch it.

## What Needs to Change

In `skills/quo-plan/SKILL.md` Step 5e's **embedded dispatch-prompt output contract** (lines ~511–530, plus the worked example):

- Extend the per-finding output contract so each finding carries **one or more enumerated fix paths**, each fix path carrying **its own depth tag** (`trivial-tweak` / `refactor-locally` / `re-architect`), alongside the existing backtick severity tag AND the existing `target:` tag. The extended finding shape:
  - finding line: `` <n>. `<severity>` target: <PRD|SDD|Plan-Bee-body|Epic:<epic-id>> <one or more fix-path lines> — <description> `` (severity stays backtick; the `target:` tag stays exactly as-is)
  - fix-path line: `(<letter>) [depth:<trivial-tweak|refactor-locally|re-architect>] <description of that fix path>`
  - Single-path findings remain valid (one `(a)` block); the shape is uniform whether 1 or N paths.
- The fix-path-line shape `(<letter>) [depth:...]` MUST match the shape used by sibling emission Epics `t1.ut9.9c` / `t1.ut9.jn` / `t1.ut9.q2` (and the new fix-path element in `t1.ut9.29`) so the Phase-2 orchestrator parser (Epic `t1.ut9.no`) derives `(num-paths, max-depth)` uniformly. Depth is emitted-but-not-yet-consumed in Phase 1 — add no routing/consumption prose for it.
- Update the worked example (lines ~519–530) so each example finding renders the existing severity + `target:` tags PLUS the new depth + fix paths. Per SR-1.2 calibration discipline, ensure at least one example per depth bucket (`trivial-tweak`, `refactor-locally`, `re-architect`) and at least one single-path and one multi-path example appear. Keep examples plan-review-flavored (PRD/SDD/Plan-Bee/Epic decomposition substance, consistent with the existing examples). **Example-placement guidance:** keep the worked-example block minimal-but-shape-complete (it doubles as a template); place the full per-bucket calibration set in the contract prose rather than bloating the single example block.
- **Preserve verbatim:** the `target:` tag contract (lines 514–515), the verdict trailer line and its semantics (lines 532–547), the clean/no-findings shape (lines 549–553), and the three verdict-conditional routing trailers **Shape A / Shape B / Shape C** (lines 561–609) — including their two-step `TaskCreate` → `AskUserQuestion` prose, finite-choice lists, Recommended markers, counter-anchor clauses, and `docs/doc-writing-guide.md` references. The new emission shape sits *inside* the numbered findings list ABOVE the verdict trailer; the trailers themselves are unchanged in form.
- **Do NOT touch** Step 5e's substantive holistic-judgment review lane (the "SUBSTANCE: critique the overall plan as a coherent solution" framing and the EXPLICITLY-OUT-OF-SCOPE prose-quality exclusions at lines ~503–509) — PRD non-goal forbids changing Step 5e substance.

## Why

Severity describes *how important fixing-at-all is*; depth describes *what fixing costs*. Step 5e's plan-review findings already carry severity + a routing `target:`; adding the depth dimension + fix-path enumeration makes its emission feed the Phase-2 orchestrator routing table on the same `(num-paths, max-depth)` basis as the four in-flow review skills. Phase 1 — depth emitted, not yet consumed; zero new user gates beyond the existing Step 5e gate; zero `b.wii` exposure.
