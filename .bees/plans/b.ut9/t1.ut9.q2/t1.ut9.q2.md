Extend `skills/quo-test-writer-review/SKILL.md` Step 4 (Generate Work Item List) so each finding emits a severity tag (`blocker` / `suggestion` / `nit`) AND one or more enumerated fix paths, each with its own depth tag (`trivial-tweak` / `refactor-locally` / `re-architect`). Symmetric to Epics 1–2. This is the *first* introduction of severity tagging to this skill — findings currently carry no severity tag.

## Scope

- Document the new output shape in the SKILL.md's output-shape section (currently at lines 150–178).
- Provide at least one worked example.
- Preserve the existing two-trailer-shape pattern.

## Acceptance criteria

1. The skill's output-shape section documents severity tags AND per-fix-path depth tags AND enumerated fix paths.
2. At least one worked example renders in the skill prose with the new shape.
3. The two existing trailer shapes are preserved verbatim.
4. A live run of `/quo-test-writer-review` against any test diff produces findings in the documented shape.

Out of scope: changes to other review skills (separate Epics), changes to `agents/test-reviewer.md` (Epic 6), changes to orchestrator routing (Epic 7).
