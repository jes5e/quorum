Add the "Accepted compromises" surface to the end-of-run summary in BOTH `skills/quo-fix-issue/SKILL.md` AND `skills/quo-execute/SKILL.md`. The summary surface reads the compromise tracker file (per Epic 8) and renders its contents under a labeled section. Per PRD FR-5 and SDD SR-3.4.

## Scope

- Extend `/quo-fix-issue` Section 7's per-Issue summary with a new "Accepted compromises" section.
- Extend `/quo-execute`'s end-of-run summary surface (Section 7 or wherever the run-end summary lives) with the analogous section.
- The section reads the tracker file at the documented path (per Epic 8), iterates entries, and renders one bullet per entry showing the finding, the chosen path, the rationale, and the follow-up Issue ID (or `none`).
- When the tracker is empty (no compromises accepted during the run), the section is OMITTED ENTIRELY — do not render an empty heading or "N/A" placeholder.
- Edge case (per SDD SR-6.5): when the tracker accumulates more than ~10 entries, surface them all in full with a short prologue noting the volume.

## Acceptance criteria

1. The end-of-run summary in both orchestrators includes the "Accepted compromises" section logic.
2. When the tracker file is empty (or doesn't exist for the current run), the section is omitted entirely.
3. When the tracker has entries, the section renders one bullet per entry with the four canonical fields.
4. A live run that triggers ≥1 compromise produces a summary with the section visible.
5. A live run with no compromises produces a summary with the section absent.

## Dependencies

Depends on Epic 8 (the tracker file must exist before the summary can read it).

Out of scope: post-completion review consumption of the tracker (Epic 10 — that's a separate consumer of the same tracker).
