## Outcome

`/quo-file-issue` redesigned to (a) drop its Step 4 doc-mutation
behavior, (b) capture doc-divergence observations in the Issue body
instead, (c) be mid-conversation aware (no redundant discovery
questions when context exists), and (d) deepen the body template with
optional rationale-preserving sections.

## Scope

1. **Strip Step 4 doc-mutation.** Remove the "Check if any project
   documentation needs updating" step from `skills/quo-file-issue/SKILL.md`
   that today edits `docs/prd.md` / `docs/sdd.md` / README at filing
   time. The remediation belongs to `/quo-fix-issue`'s doc-writer
   pass, not to filing.

2. **Add `## Doc divergence noted` body section.** Replace Step 4's
   action with an observation-capture section in the Issue body
   template. When the issue surfaces a doc claim that's wrong (or a
   doc gap), record it in this section as plain prose pointing at
   the file/section that's wrong and what's wrong about it. The
   `/quo-fix-issue` doc-writer pass (Epic 6's expanded responsibility)
   consumes this section as part of fix scope.

3. **Mid-conversation context awareness.** When `/quo-file-issue` is
   invoked from a substantive preceding conversation, skip the
   `AskUserQuestion` discovery prompt ("What's the issue?") and
   distill the conversation directly into the Issue body. Same
   detection heuristic as Epic 4's `/quo-plan` redesign.

4. **Body template extension — optional rationale sections.** The
   current template (`Description / Current behavior / Expected
   behavior / Impact / Suggested fix`) is shallow and loses analytical
   depth. Add two optional sections:
   - `## Background and rationale` — populated when the issue came
     out of substantive analysis (e.g., a thick discussion that
     identified the bug). Captures *why* this is a bug, *what root
     cause was ruled out*, *trade-offs* surfaced during analysis.
   - `## Decisions and rejected alternatives` — populated when the
     suggested-fix path was chosen over alternatives that were
     considered. Captures the alternatives and reasoning so
     `/quo-fix-issue`'s engineer doesn't re-litigate.

   These sections are OPTIONAL. Casual one-line bug reports omit
   them; analytical issues populate them.

5. **`/quo-fix-issue` doc-writer consumption.** Verify that
   `/quo-fix-issue`'s Section 5 doc-sync pass (the doc-writer
   dispatch) reads the Issue body's `## Doc divergence noted`
   section if present and applies the corrections as part of the
   fix. This may require a small prose update in
   `skills/quo-fix-issue/SKILL.md` to mention the new section
   explicitly.

## Acceptance criteria

- `/quo-file-issue` invoked on a doc-divergence bug captures the
  observation in a `## Doc divergence noted` section in the Issue
  body and leaves `docs/prd.md`, `docs/sdd.md`, and the README
  untouched. Verifiable by `git status` showing no doc changes after
  the run.
- `/quo-fix-issue` invoked on an Issue with `## Doc divergence noted`
  applies the doc correction as part of the fix scope (engineer
  fixes the code if needed; doc-writer fixes the doc).
- `/quo-file-issue` invoked from a preceding rich conversation does
  not re-ask the discovery question; it produces a draft Issue body
  based on the conversation for review.
- Issues filed from rich conversations include `## Background and
  rationale` and/or `## Decisions and rejected alternatives` sections
  as appropriate (filled with substance, not stubs).
- Casual one-line `/quo-file-issue "<description>"` invocations
  produce Issues without the optional rationale sections — the
  template stays light when there's nothing to capture.

## Implementation notes

- This Epic is independent — it doesn't depend on the Specs hive,
  the new write-prd/write-sdd skills, or the PM two-hop lookup. It
  only touches `skills/quo-file-issue/SKILL.md` and a small prose
  update in `skills/quo-fix-issue/SKILL.md`.
- The body-template change should be backward-compatible — existing
  Issues with the old template still parse fine; the new template
  just adds optional sections.
- Mid-conversation awareness uses the same detection heuristic as
  Epic 4's `/quo-plan` redesign; coordinate the prose pattern so
  the two skills look similar where they should.
