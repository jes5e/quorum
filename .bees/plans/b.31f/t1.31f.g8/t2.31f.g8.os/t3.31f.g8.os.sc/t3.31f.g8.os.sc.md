## Context

Parent Task `t2.31f.g8.os` redesigns `/quo-file-issue` to be mid-conversation aware. Today's Step 1 ("What's the issue?" via AskUserQuestion when called without arguments) treats every invocation as cold, even when invoked from a substantive preceding discussion. This Subtask adds the mid-conversation detection branch and the distill flow, and verifies the restart branch (cold invocation) is preserved unchanged.

The parent Bee `b.31f` body Problem 2 frames the rationale: information loss across the filing boundary. Mid-conversation awareness preserves analytical richness so the optional-rationale-sections Subtask can capture it in the body template.

Sibling Epic `t1.31f.y2`'s Subtask `t3.31f.y2.o7.qn` defines the same detection heuristic for `/quo-plan` Steps 1-2. The detection prose in this Subtask must align with `.qn`'s — same indicative signals, same err-toward-distill principle, same finite-choice approval-gate shape — so users get a consistent distill-vs-restart experience across the planning and filing skills. Read `.qn`'s prose first; mirror its language.

This Subtask only touches Step 1 (and inserts a new step before Step 1). Steps 2-6 are not modified.

## What Needs to Change

Edit `skills/quo-file-issue/SKILL.md` Steps region. Existing Step 1 prose is wrapped under a new restart branch; a detection step and distill branch are added before it.

1. **Insert a mid-conversation detection step before Step 1.** Add a clearly-labeled new step (e.g., `### 0. Detect Mid-Conversation Context` or renumber Steps 1-6 to 1-7). The step branches on whether the prompt window contains substantive prior context about the issue.

2. **Detection heuristic prose.** Mirror `t3.31f.y2.o7.qn`'s detection language. Indicative signals (adapted to issue-filing wording):
   - Rich back-and-forth in the same session about the bug or its scope, root cause, or alternatives considered.
   - The user's invocation message contains substantive scope information beyond a one-line title.
   - An explicit hint that this is a continuation of a debugging or analytical discussion.

3. **Document the err-toward-distill principle inline.** State explicitly that the heuristic should tilt toward distilling rather than restarting when the choice is ambiguous. Mirror `.qn`'s exact phrasing.

4. **Distill branch.** When the heuristic fires:
   - Skip the existing Step 1 "What's the issue?" `AskUserQuestion` prompt.
   - Distill the prior conversation into a draft Issue body matching the Step 3 body-template shape (Description / Current behavior / Expected behavior / Impact / Suggested fix, plus the optional `## Doc divergence noted` section established by the strip-and-divergence sibling Subtask and the optional rationale sections established by the rationale-sections sibling Subtask when applicable).
   - Present the distilled Issue body to the user via `AskUserQuestion`. Finite choices: approve / revise / cancel.
   - On approve, proceed directly to Step 2 (research / duplicate check) and Step 3 (create the ticket) with the distilled body as the starting draft.
   - On revise, iterate in prose with the user, then re-present.
   - On cancel, exit cleanly.

5. **Restart branch (cold invocation).** When the heuristic does NOT fire — solo invocation from a fresh session, or `/quo-file-issue <description>` with only a one-line description and no rich preceding discussion — wrap the existing Step 1 prose under a clearly-labeled "restart branch" header. The behavior in this branch must be identical to today's Step 1.

6. **No regression to Steps 2-6.** Step 2's research-and-duplicate-check, Step 3's ticket creation, Step 4's doc-divergence capture (as redesigned by the strip-and-divergence sibling Subtask), Step 5's commit logic, and Step 6's report-back stay.
