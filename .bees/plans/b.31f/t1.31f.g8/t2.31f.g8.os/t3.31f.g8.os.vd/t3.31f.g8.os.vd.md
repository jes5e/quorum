## Context

Parent Task `t2.31f.g8.os` redesigns `/quo-file-issue`'s body template to preserve analytical depth that comes out of substantive preceding discussion. Today's body template (`Description / Current behavior / Expected behavior / Impact / Suggested fix`) is shallow and loses that depth. This Subtask adds two OPTIONAL sections that capture analytical context when the issue came from substantive analysis.

The mid-conversation detection sibling Subtask ensures the prior conversation gets distilled rather than discarded; this Subtask gives the distillation somewhere in the body template to land its analytical content.

The optional-when-empty disambiguation is different from `/quo-write-prd`/`/quo-write-sdd`'s mandatory-always-present rule because Issues come in many sizes — one-line bug reports vs deep analytical sessions — and requiring the rationale sections always-present would force noise into casual issues. The template must make this difference explicit.

This Subtask logically follows the strip-and-divergence sibling Subtask (which establishes the body-template shape with `## Doc divergence noted` added) and the mid-conversation detection sibling Subtask (whose distill branch populates these new sections from prior-conversation content). Working it after both lands avoids merge conflicts in Step 3's body-template markdown block.

## What Needs to Change

Edit `skills/quo-file-issue/SKILL.md` Step 3 body-template block.

1. **Add `## Background and rationale` as OPTIONAL.** Extend the body-template markdown block to include this new section. Document inline that it's populated when the issue came out of substantive analysis (root cause investigation, alternative-cause ruling, trade-off discussion) and omitted on casual one-line bug reports. Captures *why* this is a bug, *what root cause was ruled out*, and *trade-offs* surfaced during analysis.

2. **Add `## Decisions and rejected alternatives` as OPTIONAL.** Extend the body-template markdown block to include this new section. Document inline that it's populated when the suggested-fix path was chosen over alternatives that were considered, and omitted otherwise. Captures the alternatives considered and the reasoning for choosing the suggested fix, so `/quo-fix-issue`'s engineer doesn't re-litigate decisions the user has already made.

3. **Document the OPTIONAL contract explicitly.** Add a paragraph (or labeled note) under the body-template block that states: these two sections — plus the `## Doc divergence noted` section established by the strip-and-divergence sibling Subtask — are OPTIONAL. They are omitted entirely (not stub-headed-then-empty) when the issue does not have content for them. This is intentionally different from the writer skills' (`/quo-write-prd`, `/quo-write-sdd`) mandatory-always-present rule because Issues come in many sizes; requiring the rationale sections always-present would force noise into casual issues. Casual one-line `/quo-file-issue "<description>"` invocations produce Issues without these sections; analytical issues populate them.

4. **Hook into the distill branch.** Update the distill-branch prose added by the mid-conversation detection sibling Subtask (or add a cross-reference if that Subtask already establishes the hand-off shape) so the distillation populates these two sections when the prior conversation contains the corresponding content. Specifically: when the prior conversation includes root-cause analysis, alternative-cause ruling, or trade-off discussion, the distilled draft body's `## Background and rationale` section captures it; when the prior conversation includes alternative-fix discussion, the distilled draft body's `## Decisions and rejected alternatives` section captures it. When the prior conversation contains neither, the sections are omitted from the draft.

5. **Hook into the restart branch.** The restart branch (cold invocation, one-line description) should NOT populate these sections — there's no analytical content to capture from a one-line description. The body template renders without them in this case, matching today's body shape plus the `## Doc divergence noted` addition (when applicable).

6. **Title-and-bundling guidelines unchanged.** Step 3's title-guidelines block and the `## House style: bundle related issues` section at the top of the SKILL.md are not modified by this Subtask.
