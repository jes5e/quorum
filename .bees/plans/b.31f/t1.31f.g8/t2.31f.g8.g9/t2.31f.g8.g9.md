## Context

Epic `t1.31f.g8` redesigns `/quo-file-issue` to add a new `## Doc divergence noted` section to the Issue body template (sibling Task 1 of this Epic). When `/quo-fix-issue` runs on such an Issue, its existing doc-sync pass (the doc-writer dispatch in Section 5 of `skills/quo-fix-issue/SKILL.md`) must consume the section and apply the doc corrections as part of the fix scope.

This Task delivers the small consumption-side update: verify that `/quo-fix-issue`'s doc-sync pass reads the `## Doc divergence noted` section if present, and add a small prose mention to `skills/quo-fix-issue/SKILL.md` if needed to make the consumption explicit.

The work is bounded — likely a few prose lines in `skills/quo-fix-issue/SKILL.md`. Sibling Task 1 owns the substantial redesign work in `skills/quo-file-issue/SKILL.md`.

## What Needs to Change

Two-step verification + minimal prose update on `skills/quo-fix-issue/SKILL.md`:

1. **Verify (read-only).** Read the existing `/quo-fix-issue` Section 5 (doc-sync pass via doc-writer dispatch). Identify whether the prose already explicitly handles the case where the Issue body contains optional sections beyond the standard `Description / Current behavior / Expected behavior / Impact / Suggested fix` template. If yes (the doc-writer dispatch reads the full Issue body and consumes whatever sections are present), the consumption is already implicit and only a small explicit mention is needed. The consumption-side update is the small wire-up that ensures the input reaches the right output.
