## Context

Epic `t1.31f.y2` redesigns `/quo-plan` to be mid-conversation aware: when invoked from a substantive preceding conversation, it should distill that conversation into the feature scope rather than restarting with discovery questions. This Task delivers that behavior in Steps 1 and 2 of `/quo-plan`'s workflow.

Today's `/quo-plan` Step 1 ("Before I start researching, is there anything I should know?") and Step 2 (numbered clarifying-questions list) treat every invocation as cold, even when invoked from a substantive preceding discussion. This Task adds a mid-conversation detection branch that skips these prompts when the prior context is rich enough to distill from.

- **Task 3** of this Epic — Plan Bee creation update (`bees`-resolver `reference_materials`), drop Scoped-marker, simplify Plan Bee body, verify Step 7.

The detection language must align across `/quo-plan`, `/quo-write-prd`, and `/quo-write-sdd` — if these three skills are invoked in sequence on the same conversation, all three must make the same distill-vs-restart decision. ## What Needs to Change

`skills/quo-plan/SKILL.md` Steps 1 and 2:

1. **Mid-conversation detection branch (new step inserted before Step 1).** Insert a detection step at the start of the workflow that branches on whether the prompt window contains substantive prior context about the feature. Detection signals (skill author's call on exact wording, but must align with sibling skills' detection prose):
   - Rich back-and-forth in the same session about the feature's scope, rationale, or alternatives.
   - The user's invocation message contains substantive scope information beyond a one-line title (e.g., a paragraph or more describing what they want).

2. **Distill branch.** When the heuristic fires:
   - Skip the Step 1 "Before I start researching, is there anything I should know?" prompt.
   - Skip the Step 2 numbered clarifying-questions list.
   - Distill the prior conversation into a scope summary (what the feature does, why, key decisions, rejected alternatives, constraints).
   - Present the distilled scope to the user via `AskUserQuestion` for approval (finite choices: approve / revise / cancel) per CLAUDE.md `## AskUserQuestion usage`.
   - On approval, proceed to Step 3 (existing Plan Bee scope-approval gate stays). On revise, iterate in prose. On cancel, exit.

3. **Restart branch.** When the heuristic does NOT fire (cold solo invocation or `/quo-plan <description>` with no rich preceding conversation), Steps 1 and 2 run as today — unchanged.

4. **Cross-skill consistency.** The detection heuristic prose must align with `t3.31f.by.7h.xa` and `t3.31f.5u.f3.pt`'s heuristics (same indicative signals, same err-toward-distill principle, same finite-choice approval-gate shape). Read those Subtasks first; mirror their language. The reason for alignment: when `/quo-plan` invokes `/quo-write-prd` and `/quo-write-sdd` in Step 4 (sibling Task 1 of this Epic), passing the distilled scope, all three skills should see the same prior context and make the same distill-vs-restart decision. If `/quo-plan` distills but `/quo-write-prd` decides to restart discovery, the user gets asked questions twice; if `/quo-plan` restarts but the writer skills distill, the user is asked questions during planning AND during writing.

5. **Document the err-toward-distill principle.** State inline in the prose that the detection heuristic should tilt toward distilling rather than restarting when the choice is ambiguous. Future maintainers must not tighten the heuristic into a stricter "only fire when X is unambiguously true" gate, which would defeat the design intent.

6. **No regression to Steps 3 onwards.** The user-facing scope-approval gate at Step 3 (the `AskUserQuestion` confirming the distilled scope) remains. Step 5c's Epic-list approval also remains. The mid-conversation branching only affects Steps 1 and 2's behavior.

## Why

Information loss across the planning boundary (parent Bee body Problem 2): a thorough planning conversation gets funneled into a 2-3-sentence Plan Bee body when discovery restarts; rationale, rejected alternatives, and constraints are lost. Mid-conversation awareness preserves that richness by distilling into the new Spec Bee's PRD and SDD child tickets (sibling Task 1 of this Epic handles the delegation; this Task ensures the planning conversation is preserved enough to distill from).

The detection-prose alignment across `/quo-plan`, `/quo-write-prd`, and `/quo-write-sdd` matters because the three skills are designed to work together. A user planning a feature mid-conversation invokes `/quo-plan`; `/quo-plan` (Task 1 of this Epic) invokes the writer skills; all three consume the same conversation context. Divergent detection logic means the user is interviewed redundantly.
