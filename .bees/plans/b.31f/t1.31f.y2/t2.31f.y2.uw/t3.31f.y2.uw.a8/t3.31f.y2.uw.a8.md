## Why

The parent Task body (`t2.31f.y2.uw` Section 5) explicitly calls out that Step 7's prose must handle the no-docs-changes case gracefully, and that the docs-changes case effectively never fires for `/bees-plan` after this Epic lands. The current prose's trailing "drop docs/ from add list" line is correct but is buried as an afterthought; making the no-docs-changes case the dominant path matches the new reality and is clearer for future maintainers (and for downstream agents reading the skill prose at runtime).

The explicit empty-`add`-list guard prevents a subtle correctness bug: if `add` arguments are constructed by string concatenation and end up empty, `git add` with no args is a no-op on most setups but `git add @addArgs` in PowerShell (where `@addArgs` is an empty array) splats to nothing and could stage everything in some shells/configs. Better to guard explicitly.
