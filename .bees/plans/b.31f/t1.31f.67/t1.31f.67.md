## Outcome

Verify (and update prose where needed) that `/bees-plan-from-specs`
continues to work unchanged for the file-based PRD/SDD path after the
Epic 1-6 changes land. Specifically: the file-resolver code path in PM
and breakdown two-hop logic must fall through cleanly, the
Scoped-marker machinery must remain valid for `--feature` mode, and the
skill prose must clearly position itself as the "I have docs on disk
already" path while `/bees-plan` is the "let's write specs as tickets"
path.

## Scope

1. **End-to-end regression test (manual).** Run
   `/bees-plan-from-specs <prd-path> <sdd-path>` against a sample
   PRD and SDD on disk and verify:
   - Plan Bee is created with
     `--reference-materials '[{"value":"<prd>"},{"value":"<sdd>"}]'`
     (no resolver field, defaults to `file-path`).
   - PM agent dispatched against this Plan Bee (e.g. via running
     `/bees-execute` partway through, or by inspecting the agent
     prompt) reads the file-resolver content correctly.
   - `/bees-breakdown-epic` Spec Traceability Review cites file
     content and section numbers correctly.

2. **Scoped-marker mode regression test.** Run
   `/bees-plan-from-specs <prd> <sdd> --feature "<title>"` against
   cumulative PRD/SDD and verify:
   - The Scoped-marker line is emitted in the Plan Bee body.
   - PM agent dispatched against this Plan Bee detects the marker,
     invokes `scoped_marker_resolver.py`, and uses the scoped
     subsection content for spec review.
   - `/bees-breakdown-epic`'s own marker handling works the same way.

3. **Skill prose update.** Update `skills/bees-plan-from-specs/SKILL.md`
   Overview section to clarify positioning relative to the redesigned
   `/bees-plan`:
   - `/bees-plan` is for "I want to design from scratch with Claude;
     specs land as tickets in the Specs hive."
   - `/bees-plan-from-specs` is for "I already have PRD and SDD on
     disk (authored by hand or by external tooling); turn them into
     a Plan Bee directly."
   - The cumulative-PRD/SDD pattern is now driven only by external
     authoring + `/bees-plan-from-specs`, not by `/bees-plan`. Note
     this explicitly so users understand the historical pattern is
     no longer auto-generated.

4. **No behavioral change to `/bees-plan-from-specs` itself.** The
   skill's logic stays as it is today. This Epic's deliverable is
   verification + prose clarification, not behavioral change.

## Acceptance criteria

- Manual run of `/bees-plan-from-specs <prd> <sdd>` (no `--feature`)
  against single-feature on-disk docs produces a working Plan Bee +
  Epics, and downstream `/bees-execute` correctly reads spec content
  via the file resolver. PM and breakdown both work end-to-end.
- Manual run of `/bees-plan-from-specs <prd> <sdd> --feature "<title>"`
  against cumulative on-disk docs produces a Plan Bee with a
  Scoped-marker line, and downstream PM/breakdown correctly scope
  spec content to the feature subsection via the marker.
- `skills/bees-plan-from-specs/SKILL.md` Overview clearly positions
  the skill relative to the redesigned `/bees-plan`.
- No behavioral changes to the skill's logic itself (`bees create-ticket`
  invocation, marker emission, doc reading) — verifiable by minimal
  diff to the skill.

## Implementation notes

- This Epic is a regression-protection Epic. The risk being mitigated:
  Epics 4-6 introduce the `bees`-resolver path; if PM or breakdown's
  two-hop logic is implemented in a way that breaks the `file-path`
  branch, `/bees-plan-from-specs`-created Plan Bees would silently
  fail or use wrong spec content.
- The verification can be either (a) manual end-to-end runs against a
  sample PRD/SDD pair, or (b) a smaller targeted check that the
  resolver-branch logic in PM and breakdown still handles `file-path`
  correctly when invoked against a Plan Bee with file-resolver
  reference_materials. Either is sufficient for this Epic's
  acceptance.

## Dependencies

Epic 5 (PM + breakdown two-hop lookup) — this Epic verifies Epic 5
didn't break the file-resolver path.

## Out of scope (for this Epic)

- Adding new features to `/bees-plan-from-specs`.
- The Scoped-marker contract itself (lives in
  `docs/doc-writing-guide.md`; not changed by this Epic).
- doc-writer cumulative-doc updates (Epic 6).
