## What Needs to Change

In `skills/bees-breakdown-epic/SKILL.md` Section 2 ("Fetch and Analyze Epic"):

1. **Polymorphic `reference_materials` reading.** The existing prose reads `reference_materials` as a list of file paths ("Read the source material linked from the parent Bee's `reference_materials`"). Update to handle the polymorphic shape: each entry has a `value` and an optional `resolver` field. Iterate the list, branch on `resolver`:
   - `resolver == "file-path"` (or absent — default): treat `value` as a path on disk, read the file content as today.
   - `resolver == "bees"`: treat `value` as a Spec Bee ticket ID; enumerate its child tickets via `bees show-ticket`, identify the PRD child and SDD child by exact title-match (`PRD` and `SDD`), and read each child's body as the spec content.
   - Mirror the exact recipe `agents/pm.md` establishes (sibling Task `t2.31f.4u.j5`) — same title-match, same `bees show-ticket` enumerate-children pattern.

2. **Scoped-marker skip-on-bees branch.** The existing Scoped-marker handling ("Check for the Scoped-marker on the parent Bee") runs `scoped_marker_resolver.py` on the parent Bee body. With the bees resolver, the Spec Bee is already feature-scoped (one Spec Bee per feature); there is no cumulative file to scope. Add a pre-branch:
   - When the resolver is `bees`, SKIP Scoped-marker resolution entirely. Document inline why (Spec Bee is already feature-scoped; no cumulative-file scoping is needed).
   - When the resolver is `file-path` (or absent — default), run the existing Scoped-marker helper invocation as today, verbatim. This preserves `/bees-plan-from-specs --feature` mode that sibling Epic `t1.31f.67` regression-tests.

3. **Plan Bee body fallback unchanged.** The existing fallback ("If `reference_materials` is null/empty, the Plan Bee body itself is the authoritative scope document") remains as today — it is the no-resolver case. Do not remove it.

4. **Project-neutral, paired snippets.** Any new shell snippets ship as paired POSIX bash + Windows PowerShell variants. Single-literal-command Bash etiquette applies. No hardcoded language-specific commands or this-repo paths.
