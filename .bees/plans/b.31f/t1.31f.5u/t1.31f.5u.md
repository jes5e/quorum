## Outcome

A new `/bees-write-sdd` skill that authors or revises an SDD ticket as a
`t1=Doc` child of a Spec Bee. Mirror of Epic 2 but for Software Design
Document content. Composable: invokable solo for SDD revisions, also
invoked inline by the redesigned `/bees-plan` (Epic 4).

## Scope

1. **New skill at `skills/bees-write-sdd/SKILL.md`.** Adapt apiary's
   `skills/write-srd/skill.md` as the starting point. **Rename
   SRD → SDD** for consistency with our existing Documentation Locations
   contract key (`Internal architecture docs (SDD)` in CLAUDE.md). All
   prose in the new skill should refer to SDD; the title of the
   resulting child ticket is `SDD`.

2. **Required sections in the SDD ticket body** (extending apiary's set):
   - Codebase exploration findings (architecture, affected modules,
     existing patterns, data models, test fixtures, configuration —
     apiary's required scaffolding is right)
   - Numbered SR-style requirement groups, organized by domain/concern
   - Test Fixtures section (apiary requires this — we keep it; prevents
     hardcoded values from sneaking past test-helper conventions)
   - Existing Behavior (what must NOT change)
   - Documentation (what docs must be written after implementation)
   - **Background and rationale** (NEW — same reasoning as Epic 2)
   - **Decisions and rejected alternatives** (NEW — captures architectural
     decisions and the trade-offs considered, so downstream cold-session
     agents don't re-litigate design choices)

3. **Mid-conversation context awareness.** Same as Epic 2 — detect rich
   preceding context and distill instead of restarting discovery.

4. **Solo invocation.** `/bees-write-sdd <spec-bee-id>` creates the
   SDD child if it doesn't exist, OR updates it if it does. Idempotent.

5. **Inline invocation via Skill tool.** When `/bees-plan` (Epic 4)
   invokes this skill, it passes the Spec Bee ID and conversation context.

## Acceptance criteria

- `/bees-write-sdd <spec-bee-id>` invoked solo against an existing
  Spec Bee creates a `t1=Doc`-titled `SDD` child with all required
  sections present.
- Re-invoking on the same Spec Bee updates the existing SDD ticket
  rather than creating a duplicate.
- The SDD body ALWAYS includes `Background and rationale` and
  `Decisions and rejected alternatives` sections.
- The SDD's codebase-exploration content references real module names
  and patterns from the actual codebase (apiary's quality bar).
- When invoked from a preceding rich conversation, the skill does not
  restart discovery; it produces a draft based on prior context.
- Skill is invokable both directly and via the Skill tool from
  `/bees-plan` (Epic 4 verifies inline path).

## Implementation notes

- Apiary's `/write-srd` uses the Explore agent for codebase research.
  We have access to the same agent (`subagent_type=Explore`); use it
  the same way.
- Apiary's "RESEARCH NEEDED" flag for ambiguous spots is a useful
  pattern; carry it over.
- Use `--body-file <path>` per the scratch-file convention.
- Update the skill catalog table in README.md to include
  `/bees-write-sdd`.
- Note the SRD→SDD rename in the skill's prose so future readers of
  apiary docs aren't confused.
