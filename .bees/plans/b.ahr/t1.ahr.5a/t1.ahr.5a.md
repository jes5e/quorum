## Description

Extend `skills/quo-file-issue/SKILL.md` to auto-detect URL positional arguments (`^https?://`) and route them to the existing External-reference branch ŌĆö no `--reference` / `--from-github` flag required. Add an inline-invocation contract section so `/quo-fix-issue` can dispatch `/quo-file-issue` programmatically in Epic 2. Add dedupe-by-`reference_materials.value` to the External-reference branch.

This is the foundational Epic. After it lands, `/quo-file-issue <url>` works as a user-typed slash command, and the Skill-tool dispatch surface is published for Epic 2 to consume. `/quo-fix-issue` is unchanged at the end of this Epic.

## What needs to change

1. **Mode-fork extension** (`skills/quo-file-issue/SKILL.md` lines 52-61). Add a third routing input ŌĆö a positional argument matching `^https?://` is a URL token that routes to the External-reference branch. Existing flag-form routing (`--reference`, `--from-github`) continues to work and converges on the same branch.
2. **Frontmatter and Usage** (line 4 and lines 25-32). Update `argument-hint` to include the bare-URL form. Update the `### Usage` section to list the bare-URL invocation alongside the flag forms.
3. **Inline-invocation contract section** (new section near the bottom of the file, mirroring `skills/quo-write-prd/SKILL.md`'s `## Inline invocation via the Skill tool`). Document:
   - **Input shape:** free-text `args` carrying the URL (e.g., `url: <url>`).
   - **Output shape:** `issue_ticket_id`, `issue_status` (`open` on success), `action` (`created` or `reused-existing`).
   - **Behavioral guarantees:** user-facing `AskUserQuestion` gates still fire on the inline path; caller does NOT short-circuit.
4. **Dedupe by `reference_materials.value`** (extend Step 2 duplicate-check, or add a parallel sub-step inside the External-reference branch). Query open Issues for one whose `reference_materials.value` matches the URL exactly. On hit, surface via `AskUserQuestion` with `Use existing` / `File new` / `Cancel` choices, including the matched ticket's ID, title, and status in the prompt body. Choose the one-step path (if the bees CLI supports `reference_materials` as a stage-filter clause ŌĆö verify at implementation time) or the two-step path (freeform-query for `[type=bee, hive=issues, status=open]` returning `[ticket_id]`, then batch `bees show-ticket --ids ...` and filter client-side on `reference_materials[*].value`). The bees CLI rejects `reference_materials` as a `report` clause field ŌĆö confirmed empirically ŌĆö so a single freeform-query with `reference_materials` in the report is NOT viable.

The flags become silent no-op aliases on top of detection ŌĆö the `--reference` / `--from-github` documentation stays in the SKILL.md prose for backward compat and discoverability.

## Acceptance Criteria

- `/quo-file-issue https://github.com/owner/repo/issues/123` produces an Issue identical in shape to today's `/quo-file-issue --reference https://github.com/owner/repo/issues/123`: thin body with `external reference:` line, `reference_materials: [{"value":"...","resolver":"github-issue"}]`, status=`open`.
- `/quo-file-issue --reference <url>` and `/quo-file-issue --from-github <url>` continue to produce identical results to the bare-URL path.
- Re-running `/quo-file-issue <url>` against an existing matching open Issue surfaces a dedupe `AskUserQuestion`; the prompt body includes the matched ticket's ID, title, and status; choosing `Use existing` returns the matched ticket ID without filing a new one; choosing `File new` files anyway; choosing `Cancel` exits cleanly with no partial state.

- All existing invocation forms (`/quo-file-issue` interactive, `/quo-file-issue <description>`, both flag forms) continue to work without modification.
- `/quo-fix-issue` is NOT yet URL-aware at the end of this Epic ŌĆö that ships in Epic 2. The contract section enables the Epic 2 wiring but is not yet consumed.
- Skill-prose changes pass project design rules: no language-specific commands (rule 1), POSIX + PowerShell pairing for any new shell snippets (rule 2), project-neutral phrasing (rule 3), scratch-file convention preserved.
