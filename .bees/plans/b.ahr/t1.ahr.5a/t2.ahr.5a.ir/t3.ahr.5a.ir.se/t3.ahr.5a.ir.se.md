## Context

The sibling subtask updates the Mode fork in `skills/quo-file-issue/SKILL.md` to route a bare-URL positional argument through the existing External-reference branch. This subtask is the consistency follow-up: it surfaces the new bare-URL form in the user-visible advertisement (the `argument-hint` frontmatter and the `### Usage` section) and updates the two downstream cross-references that currently describe the External-reference branch as "reached only when `--reference` / `--from-github` is set" so they accurately describe all three entry shapes.

Preconditions: the sibling subtask establishing bare-URL routing in the Mode fork must land first so the new wording here is coherent with the routing semantics it describes.

## What Needs to Change

In `skills/quo-file-issue/SKILL.md`:

1. **`argument-hint` frontmatter (line 4).** Update the hint string to surface the bare-URL invocation form alongside the existing `[<description> | --reference <url> | --from-github <url>]` choices. Add `<url>` as a fourth alternation, e.g., `[<description> | <url> | --reference <url> | --from-github <url>]`. Keep the form short — it is a tab-completion hint, not a full spec.
2. **`### Usage` section (lines 25-32).** Add a bullet for the bare-URL invocation alongside the existing four bullets, with one example. Suggested placement: between the free-text-description bullet (line 30) and the `--reference` bullet (line 31), since the bare-URL form is conceptually a friendlier shorthand for `--reference`. Example bullet text: `
- `/quo-file-issue <url>` — bare-URL shorthand for external-reference mode: routes to the same External-reference branch as `--reference <url>`. A positional token starting with `http://` or `https://` is auto-detected; URLs inside a quoted description string are not.`
3. **External-reference branch heading (line 219).** Update the parenthetical from `(reached only when `--reference` / `--from-github` is set)` to also acknowledge the bare-URL positional, e.g., `(reached when `--reference` / `--from-github` is set, or when the positional argument is a URL)`.
4. **Step 5 report-back bullet (line 340).** Update the phrase `If the issue was filed in external-reference mode (`--reference` / `--from-github`)` to mention all three entry shapes, e.g., `If the issue was filed in external-reference mode (bare URL, `--reference`, or `--from-github`)`. The Step 5 user-visible summary itself does not need to differentiate which entry shape was used — the resolver name and URL reported are identical.

Do **not** modify the External-reference branch body sub-steps A, B, C beyond the heading change in (3). Do **not** touch the Mode fork section itself (covered by the sibling subtask).

Project design rules apply: no language-specific commands (rule 1 — N/A), POSIX + PowerShell pairing for any new shell snippets (rule 2 — N/A, no shell snippets added), project-neutral phrasing (rule 3 — the bare-URL form is universal across every host project), scratch-file convention preserved.
