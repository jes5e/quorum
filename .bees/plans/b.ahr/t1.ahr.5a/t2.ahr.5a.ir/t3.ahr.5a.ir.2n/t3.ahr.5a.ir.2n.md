## Context

`/quo-file-issue`'s Mode fork (in `skills/quo-file-issue/SKILL.md`, lines 52-61) currently routes only flag-form invocations (`--reference <url>`, `--from-github <url>`) to the External-reference branch; a bare-URL positional argument is mishandled as a free-text description. This subtask is the load-bearing semantic change: it teaches the Mode fork to recognize a `^https?://` positional argument and route it to the existing External-reference branch with the same effect as the flag forms. The branch body (sub-steps A, B, C starting at line 219) is unchanged — only the routing prose at the top of `## Steps` is touched.

Preconditions: none beyond the existing skill structure. This subtask is the prerequisite for the sibling subtask that advertises the new form in the frontmatter and Usage section.

## What Needs to Change

In `skills/quo-file-issue/SKILL.md`, update the `### Mode fork — in-conversation capture vs external-reference` section (lines 52-61) so the routing rule recognizes three input shapes, not two:

1. Add a third bullet to the bullet list at lines 56-57 covering the bare-URL positional form. The bullet should state the detection rule precisely: a positional token that starts with `http://` or `https://` after argument tokenization is treated as the URL value and routed to the External-reference branch, identical in effect to `--reference <url>`.
2. Add an explicit non-detection clause covering URLs embedded inside a quoted description string (e.g., `/quo-file-issue "Fix the bug at https://example.com/foo"`). The clause must state that a URL inside a quoted free-text token continues to be treated as part of the description, not auto-detected. Tokenization happens before URL detection — only standalone tokens that begin with `http://` or `https://` are URL-shaped positionals.
3. Update the existing prose at lines 59-61 that gates the External-reference branch on "any external-reference flag is present" so it instead gates on "any external-reference flag is present, OR the positional argument is a URL-shaped token". The flag forms remain accepted as silent no-op aliases on top of the bare-URL detection — their existing prose stays for backward compat and discoverability.
4. Do **not** change the External-reference branch body (sub-steps A, B, C — lines 219-291). The dedupe-on-`reference_materials.value` work is a sibling Task in this Epic and is out of scope here.
