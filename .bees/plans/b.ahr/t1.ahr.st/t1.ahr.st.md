## What needs to change

1. **Token classifier extension** (`skills/quo-fix-issue/SKILL.md` Section 1, lines 48-67). Recognize tokens matching `^https?://` as a fifth token-shape category alongside zero / `all` / single-ID / list-of-IDs.
2. **URL-resolution sub-step** (new sub-step in Section 1). Runs AFTER `#### Choose agent model preference` (lines 69-78) and `#### Validate isolation strategy` (lines 80-102) but BEFORE the existing `bees show-ticket --ids <id1> ...` upfront-validation pass at line 59. For each URL token: dispatch `/quo-file-issue` via the Skill tool with the URL as the bare-positional argument; consume the returned `issue_ticket_id` from the structured return shape; substitute the URL token in place in the working list with the resolved ID. Skill-tool dispatch failures (errors, user-cancel at one of `/quo-file-issue`'s gates) follow the existing soft-fail-on-bad-ID pattern from line 59 — drop the failed URL, continue with remaining tokens, error only when nothing valid remains.
