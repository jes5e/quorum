## Context

This Task ships the customer-visible cumulative-doc updates for the auto-detect-URLs feature. The work was deferred from Epic 1 to this Epic so the customer-visible feature doc-state is consistent — no half-shipped middle state where `/quo-file-issue` URL handling has landed but the docs and README still describe the flag-required model. ## Why

The cumulative docs are the authoritative customer/architecture-facing surface for the feature. Without the `### Feature:` subsections, future agents (Engineer, PM, Doc Writer in subsequent features) lack a documented architectural baseline to reason against; without the README updates, users discover the new invocation forms only by reading the SKILL.md prose, which is friction. Bundling all three doc updates into a single Task (and a single commit) keeps the customer-visible doc-state consistent — there's no transient state where SDD/PRD describe the new feature but README still says flag-required, or vice versa.
