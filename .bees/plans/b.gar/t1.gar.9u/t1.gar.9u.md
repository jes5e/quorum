# Epic A — Layer 1: helper unit tests

**Status at completion.** `pytest skills/` passes from a clean clone, exercising every bundled Python helper with its JSON contract. No regression in any other layer.

## Description

Set up pytest infrastructure for the repo and write per-helper unit-test files that exercise every bundled helper script's public surface. By execution time, the helpers include:

- `skills/bees-setup/scripts/file_list_resolver.py`
- `skills/bees-setup/scripts/detect_fast_path.py`
- `skills/bees-execute/scripts/force_clean_team.py`
- `skills/bees-execute/scripts/check_agent_teams.py`
- `skills/_shared/scripts/ticket_backend.py` (introduced by b.9xr Epic A)

Tests live as siblings of the helpers (`skills/<skill>/scripts/test_<helper>.py` and `skills/_shared/scripts/test_ticket_backend.py`). Pytest auto-discovery picks them up via `pytest skills/`. Tests use `tmp_path` for filesystem isolation and `monkeypatch` for environment overrides. The dispatcher's tests exercise both backend branches when the relevant CLI is present and skip cleanly otherwise (`pytest.skip` with a message naming the missing CLI).

Coverage targets: every public function, every error path, every JSON contract field documented in the helper's docstring. No mocking of the backend CLIs themselves at this layer — those are exercised in Epic C's equivalence harness.

If a pytest configuration file is needed (e.g., `pyproject.toml` or `pytest.ini`) to declare test-discovery rules or exclude scripts that aren't tests, add it to the repo root.

## Scope

- Add pytest as a development dependency (document in CONTRIBUTING.md or a `requirements-dev.txt`; this repo has no `pyproject.toml` today).
- Write `test_<helper>.py` for each of the 5+ helpers listed above.
- Each test file covers: happy path, error paths, edge cases (empty input, missing files, malformed JSON), JSON contract field-by-field assertions.
- For `ticket_backend.py`, exercise both `--backend bees` and `--backend beads` paths, skipping the unavailable backend cleanly with `pytest.skip`.
- Add a `Makefile` `test-helpers` target that runs `pytest skills/` (top-level `make test` lands in Epic C).
- Verify `pytest skills/` passes locally on this machine.

## Acceptance criteria

1. **Demo: clean run.** Agent runs `pytest skills/` from the repo root on a clean checkout and the output ends with all tests passing (zero failures, zero errors).
2. **Demo: coverage check.** Agent shows that `find skills -name 'test_*.py'` returns one file per bundled helper, and `pytest --collect-only skills/` lists at least one test per helper script.
3. **Demo: tmp-path isolation.** Agent runs the suite twice in a row and confirms no test artifacts leak between runs (no stale `.bees/` or `.beads-plans/` directories left behind).
4. **Demo: skip-cleanly behavior.** On a machine where `bd` is not installed, agent runs `pytest skills/_shared/scripts/test_ticket_backend.py` and confirms beads-branch tests SKIP (not FAIL) with a clear message naming the missing CLI.
5. **Codebase green.** `python -m pyflakes skills/*/scripts/*.py skills/_shared/scripts/*.py` still exits 0.
